## 4. Strategi Pengisian KB Gap

### Opsi A — Tunda KB, pakai patch v1.1 sebagaimana adanya

- Sprint 4 orkestrasi tetap jalan dengan worker-agen yang sudah ABD-compliant.
- Heuristik default di patch v1.1 cukup untuk fallback.
- Risiko: confidence skor rata-rata mungkin di kisaran 65–75% saja.

### Opsi B (rekomendasi) — KB Tinggi dulu, paralel dengan Sprint 4

- Bangun 3 KB Tinggi (AHSP, AMDAL/UKL-UPL, Permen PUPR PBJ) **paralel** dengan Sprint 4 P1–P2.
- Workers TEKNIK, ENVIRA, KONTRAK akan upgrade saat KB live.
- Confidence skor rata-rata bisa naik ke 75–85%.

### Opsi C — Full KB sebelum Sprint 4

- Selesaikan 11 KB dulu, baru aktivasi Sprint 4.
- Hasil confidence terbaik tapi delay Sprint 4 ~6–8 minggu.

## 5. Schema Capability Registry sebagai Database Notion

Saran: bikin database Notion **"KONSTRA Capability Registry"** sebagai single source of truth yang dapat di-query oleh manusia & agen. Schema:

| **Property** | **Type** | **Isi** |
| --- | --- | --- |
| Nama Agent / Tool | Title | PROXIMA, KONTRAK, KB-EVM, ... |
| Tipe | Select | Agent / Calculator / KB / Coordinator |
| Domain | Multi-select | Project Mgmt, Engineering, K3, ... |
| Patch Status | Select | v1.0 / v1.1 ready / v1.1 deployed / Tested |
| Triggers OUT | Relation → Capability Registry | Self-referential; agen yg dipanggil |
| Triggers IN | Relation → Capability Registry | Two-way relation |
| KB Dependencies | Relation → KB pages | KB yang dirujuk |
| Sprint Active | Multi-select | Sprint 1, 2, 3, 4 |
| Last Audit | Date | tanggal field test terakhir |
| Confidence Avg | Number | rata-rata dari telemetry |
