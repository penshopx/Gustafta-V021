## 3. Step-by-Step Deploy per Agen

Ulangi langkah ini untuk **setiap agen** sesuai urutan wave di atas:

1. **Buka** konfigurasi agen target di Gustafta Builder.
2. **Snapshot** system prompt lama (copy ke file backup `agent-XXX-prompt-v1.0-backup.txt` di local).
3. **Buka** patch source (link di tabel atas) di Notion.
4. **Copy** seluruh blok `text` di section *PATCH — System Prompt AGENT-XXX (v1.1)*.
5. **Replace** isi system prompt agen di Gustafta Builder dengan blok yang dicopy. *(Atau append jika ingin pertahankan instruksi spesifik tambahan; pastikan tidak terjadi duplikasi misi/identitas.)*
6. **Save** konfigurasi agen.
7. **Sanity check:** buka mode chat dengan agen tersebut, kirim pesan: "Test koneksi. Sebut nama dan misi singkat Anda." — verifikasi nama, misi, dan referensi ABD muncul di output.
8. **Jalankan smoke test** sesuai tabel Wave (Section 2).
9. **Isi** Field Test Tracker (Section 5) dengan hasil.
10. **Bila smoke test gagal acceptance criteria** → jalankan Rollback Procedure (Section 6) dan log issue.

## 4. Acceptance Criteria (per smoke test)

Smoke test agen dianggap **LULUS** bila:

- [ ]  **AC-1.** Output mengikuti struktur ABD-7 (8 section: Konteks, Confidence+Breakdown, Jawaban Inti, Asumsi, Inter-Agent Input, Gap, Rekomendasi, Sitasi, Closing untuk akurasi).
- [ ]  **AC-2.** Confidence Score 0–100 muncul + breakdown per komponen.
- [ ]  **AC-3.** Minimum 1 tag `[ASUMSI: ... | dasar: ...]` muncul di output.
- [ ]  **AC-4.** Tidak ada anti-pattern blocking (no "Mohon lengkapi data X dulu" sebagai jawaban utama).
- [ ]  **AC-5.** Minimum 1 sitasi regulasi/standar muncul.
- [ ]  **AC-6.** Closing line "Untuk akurasi <X>% → <Y>%, lengkapi: ..." muncul.
- [ ]  **AC-7.** Match dengan expected output di test case ≥ 70% (subjective by reviewer).

**Skor lulus = minimum 6 dari 7 AC ✅.** Bila < 6 → Rollback + log issue.

## 6. Rollback Procedure

Bila smoke test gagal AC-criteria atau ditemukan defect kritis:

1. **Stop** deployment wave berikutnya. Jangan lanjut ke agen lain dulu.
2. **Restore** system prompt agen target dari file backup snapshot (langkah 2 di Section 3).
3. **Save** konfigurasi agen.
4. **Verify** rollback berhasil dengan kirim pesan test "Sebut nama dan misi." — output harus match v1.0 lama.
5. **Log issue** di Section 7 dengan detail: agen, test ID, AC mana yang gagal, sample output yang fail, root cause hypothesis.
6. **Triage:** issue minor (AC-7 saja) → patch micro-fix; issue major (AC-1–6 ≥ 2 fail) → audit ulang patch source.
