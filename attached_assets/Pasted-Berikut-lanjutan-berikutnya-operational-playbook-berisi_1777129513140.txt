# 135. Import Validation Pipeline

```yaml id="import_validation_pipeline"
import_validation_pipeline:
  purpose: >
    Memastikan data katalog SKK yang diimpor ke sistem bersih, konsisten,
    dan siap digunakan chatbot.

  pipeline_steps:

    step_1_read_source:
      description: "Baca file CSV, spreadsheet, atau input tabel."
      required_columns:
        - "subklasifikasi"
        - "jabatan_kerja"
        - "jenjang_jabatan_kerja"
        - "kualifikasi"
        - "kkni_level"
        - "standard_code"
        - "official_link"

    step_2_normalize_text:
      actions:
        - "Trim leading/trailing spaces."
        - "Convert multiple spaces to single space."
        - "Normalize dash variants."
        - "Normalize SKKNI/SKKK capitalization."
        - "Standardize KKNI as integer."

    step_3_validate_required_fields:
      checks:
        - "subklasifikasi must not be empty."
        - "jabatan_kerja must not be empty."
        - "jenjang_jabatan_kerja must not be empty."
        - "kkni_level must be present."
        - "standard_code must be present."

    step_4_validate_allowed_values:
      kkni_level_allowed: [3, 4, 5, 6, 7, 8, 9]
      kualifikasi_allowed:
        - "Operator"
        - "Teknisi/Analis"
        - "Ahli"

    step_5_detect_duplicates:
      duplicate_key:
        - "jabatan_kerja"
        - "jenjang_jabatan_kerja"
        - "kkni_level"
        - "standard_code"
      action_if_duplicate: "flag_for_review"

    step_6_generate_role_id:
      format: "role_{slug_jabatan}_{slug_jenjang}_kkni_{kkni_level}"
      example: "role_quality_engineer_quality_engineer_kkni_6"

    step_7_publish_or_review:
      if_no_errors: "publish_to_role_catalog"
      if_warnings: "send_to_admin_review"
      if_errors: "reject_import_and_show_report"
```

# 146. Final Implementation Snapshot

```text id="final_implementation_snapshot"
Pada tahap ini, chatbot SKK Coach — Manajemen Pelaksanaan sudah memiliki:

1. Katalog jabatan kerja berbasis subklasifikasi.
2. Pencarian jabatan berdasarkan nama, KKNI, dan SKKNI/SKKK.
3. Alur pemetaan jalur berdasarkan pengalaman pengguna.
4. Modul pembelajaran per area kompetensi.
5. Asesmen mandiri dengan skor 0-4.
6. Studi kasus proyek konstruksi.
7. Simulasi wawancara berbasis STAR.
8. Checklist bukti kompetensi.
9. Glossary dan FAQ.
10. Fallback dan disambiguation.
11. Safety controls.
12. API design.
13. Database schema.
14. Admin CMS specification.
15. Testing, analytics, audit log, dan deployment checklist.

MVP yang paling disarankan untuk dibangun terlebih dahulu:
- main menu,
- role lookup,
- KKNI lookup,
- standard lookup,
- role disambiguation,
- asesmen mandiri dasar,
- checklist bukti,
- fallback aman.

Setelah MVP stabil, tambahkan:
- studi kasus lanjutan,
- simulasi wawancara,
- adaptive learning,
- admin dashboard,
- analytics dashboard,
- export ringkasan belajar.
```
