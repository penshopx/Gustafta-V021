# 🧠 CIVILPRO – KPI ANALYTICS ENGINE (Gedung Flagship)

## 🎯 Tujuan

1. User input data KPI bulanan
2. CIVILPRO:

   * Hitung interpretasi
   * Identifikasi risiko
   * Tandai merah/kuning/hijau
   * Beri rekomendasi manajerial
   * Tawarkan konversi ke bahan uji kompetensi

---

# 🔷 STEP 1 – FORMAT INPUT DATA (Sederhana)

CIVILPRO meminta user isi seperti ini:

```yaml
DATA_KPI_BULANAN:
  luas_gedung_m2: 15000
  total_kwh: 180000
  downtime_lift_jam: 20
  total_jam_operasional: 720
  realisasi_pm_percent: 88
  jumlah_keluhan: 35
  sla_vendor_percent: 82
```

User bisa isi manual. Tidak perlu file Excel dulu.

---

# 🔷 STEP 2 – ANALISIS OTOMATIS CIVILPRO

CIVILPRO menghitung:

* kWh/m²
* Downtime %
* Bandingkan dengan target
* Identifikasi risiko
* Klasifikasi warna

---

# 📊 CONTOH OUTPUT ANALISIS

```yaml
KPI_ANALYSIS:
  kwh_per_m2: 12 kWh/m2
  status_energi: "Kuning (di atas rata-rata target 10 kWh/m2)"
  
  downtime_lift_percent: 2.7%
  status_lift: "Merah (melewati batas <2%)"
  
  realisasi_pm: 88%
  status_pm: "Kuning (target ≥95%)"
  
  keluhan: 35
  status_keluhan: "Merah (indikasi penurunan service quality)"
  
  sla_vendor: 82%
  status_vendor: "Merah (vendor tidak terkendali)"
```

---

# 🔥 STEP 3 – INTERPRETASI STRATEGIS

CIVILPRO tidak berhenti di angka.

Dia lanjut:

### 🚨 Risiko Operasional

* PM rendah → potensi gangguan meningkat
* SLA rendah → vendor tidak efektif
* Downtime tinggi → risiko reputasi

### 💰 Risiko Finansial

* Energi tinggi → pemborosan biaya
* Downtime → potensi penalti tenant

### 📉 Risiko Reputasi

* Keluhan meningkat → tenant dissatisfaction

---

Level 90 Hari:
