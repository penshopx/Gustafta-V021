## LANJUTAN 7 — Verification & Assessor Form Engine

Modul ini membuat chatbot bisa berperan sebagai **asisten pra-verifikasi**: bukan hanya menanyakan dokumen, tetapi juga mengelompokkan dokumen sesuai item penilaian LSBU/asesor. Dalam SK Dirjen Binakon No. 37/KPTS/DK/2025, penilaian kesesuaian mencakup verifikasi-validasi, penilaian kemampuan badan usaha, dan rekomendasi; hasilnya dipetakan ke format penjualan tahunan, keuangan, TKK, peralatan, dan SMAP. 

```yaml
# -----------------------------------------------------------------------------
# LANJUTAN 7 — VERIFICATION & ASSESSOR FORM ENGINE
# -----------------------------------------------------------------------------

verification_assessor_engine:
  id: verification_assessor_engine_v1
  title: "Mesin Pra-Verifikasi dan Form Asesor SBU"
  purpose:
    - membantu pengguna mengelompokkan dokumen sesuai item penilaian SBU
    - membantu admin/LSBU internal membuat ringkasan pra-verifikasi
    - membantu mendeteksi data yang kosong, tidak selaras, atau perlu klarifikasi
    - menghasilkan status awal: Memenuhi, Perlu Perbaikan, Perlu Klarifikasi, atau Tidak Siap
  limitation: >
    Mesin ini tidak menggantikan penilaian asesor LSBU. Hasilnya hanya untuk
    pra-verifikasi dan persiapan data sebelum proses formal.
```

## 3. Flow Penjualan Tahunan / Pengalaman

## 8. Decision Engine

```yaml
verification_decision_engine:
  status_labels:
    memenuhi:
      label: "Memenuhi secara awal"
      meaning: "Data utama tersedia dan tampak selaras untuk dikonsultasikan."
    perlu_perbaikan:
      label: "Perlu perbaikan"
      meaning: "Ada data yang belum lengkap atau belum sesuai."
    perlu_klarifikasi:
      label: "Perlu klarifikasi"
      meaning: "Data tersedia tetapi perlu penjelasan/validasi tambahan."
    tidak_siap:
      label: "Tidak siap"
      meaning: "Dokumen utama tidak tersedia atau tidak selaras."

  rules:
    annual_sales:
      memenuhi:
        if:
          - "contract_available == Ada"
          - "major_item_match == Sesuai"
          - "used_before == Belum"
      perlu_klarifikasi:
        if:
          - "major_item_match == Belum yakin"
          - "used_before == Belum tahu"
      perlu_perbaikan:
        if:
          - "contract_available == Sebagian"
      tidak_siap:
        if:
          - "contract_available == Tidak ada"
          - "major_item_match == Tidak sesuai"

    financial:
      memenuhi:
        if:
          - "doc_type in ['Neraca internal', 'Laporan audit KAP']"
          - "equity_value is not empty"
      perlu_klarifikasi:
        if:
          - "currency == Mata uang asing"
          - "target_qualification == Belum tahu"
      perlu_perbaikan:
        if:
          - "doc_type == Belum tahu"
      tidak_siap:
        if:
          - "doc_type == Belum ada"

    tkk:
      memenuhi:
        if:
          - "pjbu == Sudah"
          - "pjtbu == Sudah sesuai"
          - "pjskbu == Sudah"
          - "subclassification_match == Sesuai"
          - "level_match == Sesuai"
      perlu_klarifikasi:
        if:
          - "pjtbu == Ada tapi belum yakin sesuai"
          - "subclassification_match == Belum yakin"
          - "level_match == Belum yakin"
      perlu_perbaikan:
        if:
          - "pjskbu == Sebagian"
      tidak_siap:
        if:
          - "pjbu == Belum"
          - "pjtbu == Belum ada"
          - "pjskbu == Belum"

    equipment:
      memenuhi:
        if:
          - "required == Tidak"
          - "or list_status == Lengkap and match == Sesuai"
      perlu_klarifikasi:
        if:
          - "required == Belum tahu"
          - "match == Belum yakin"
          - "registration == Tidak tahu"
      perlu_perbaikan:
        if:
          - "list_status == Sebagian"
          - "registration == Sebagian"
      tidak_siap:
        if:
          - "required == Ya"
          - "list_status == Belum ada"

    smap:
      memenuhi:
        if:
          - "type in ['ISO 37001:2016', 'PANCEK KPK', 'Dokumen penerapan SMAP']"
          - "validity == Ya"
      perlu_klarifikasi:
        if:
          - "validity == Belum tahu"
          - "implementation_record == Sebagian"
      perlu_perbaikan:
        if:
          - "type == Surat komitmen SMAP"
      tidak_siap:
        if:
          - "type == Belum ada"
```

## 12. Prompt Tambahan untuk Bot

```yaml
runtime_prompt_addition_verification:
  text: >
    Saat pengguna meminta pengecekan dokumen SBU, gunakan lima area penilaian:
    penjualan tahunan, kemampuan keuangan, TKK, peralatan, dan SMAP. Berikan status
    awal dengan label “Memenuhi secara awal”, “Perlu Perbaikan”, “Perlu Klarifikasi”,
    atau “Tidak Siap”. Jangan menyatakan lulus/gagal sertifikasi. Jika data belum
    lengkap, sebutkan dokumen yang perlu dilengkapi. Jika ada risiko ketidaksesuaian
    KBLI, subklasifikasi, pengalaman, TKK, peralatan, atau SMAP, beri peringatan
    secara jelas dan arahkan pengguna untuk validasi ke LSBU/OSS/SIJK.
```

Bagian ini membuat chatbot punya **mesin pra-verifikasi seperti checklist asesor**, sehingga alurnya menjadi:

**Cari SBU → Cek KBLI → Cek Kualifikasi → Cek Dokumen → Pra-Verifikasi 5 area → Rekomendasi Perbaikan → Ringkasan Pengajuan.**
