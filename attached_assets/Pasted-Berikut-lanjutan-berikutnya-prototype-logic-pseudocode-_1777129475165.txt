# 97. Pseudocode — Main Message Handler

```python id="main_message_handler"
def handle_user_message(user_id, user_message, session_state):
    normalized_text = normalize_input(user_message)

    intent_result = detect_intent(normalized_text)
    entities = extract_entities(normalized_text)

    session_state["last_user_input"] = user_message
    session_state["last_intent"] = intent_result.intent

    if intent_result.intent == "greet":
        return render_onboarding(session_state)

    if intent_result.intent == "lookup_role":
        return handle_role_lookup(entities, session_state)

    if intent_result.intent == "lookup_kkni":
        return handle_kkni_lookup(entities, session_state)

    if intent_result.intent == "lookup_standard":
        return handle_standard_lookup(entities, session_state)

    if intent_result.intent == "compare_roles":
        return handle_role_comparison(entities, session_state)

    if intent_result.intent == "start_assessment":
        return start_assessment_flow(entities, session_state)

    if intent_result.intent == "answer_assessment_question":
        return handle_assessment_answer(user_message, session_state)

    if intent_result.intent == "start_learning":
        return start_learning_module(entities, session_state)

    if intent_result.intent == "start_scenario":
        return start_scenario_flow(entities, session_state)

    if intent_result.intent == "answer_scenario":
        return evaluate_scenario_answer(user_message, session_state)

    if intent_result.intent == "start_mock_interview":
        return start_mock_interview(entities, session_state)

    if intent_result.intent == "answer_mock_interview":
        return evaluate_mock_interview_answer(user_message, session_state)

    if intent_result.intent == "glossary_lookup":
        return handle_glossary_lookup(entities, session_state)

    return handle_fallback(user_message, session_state)
```

```python id="role_lookup_handler"
def handle_role_lookup(entities, session_state):
```

```python id="assessment_engine"
def start_assessment_flow(entities, session_state):
    target_area = entities.get("subklasifikasi") or infer_area_from_target_role(session_state)

    if not target_area:
        return {
            "text": "Pilih area asesmen mandiri yang ingin Anda mulai.",
            "quick_replies": [
                "K3/Keselamatan",
                "Manajemen Proyek",
                "Mutu",
                "Kontrak",
                "Biaya/QS"
            ]
        }

    assessment = load_assessment_by_area(target_area)

    session_state["current_flow"] = "assessment"
    session_state["assessment_id"] = assessment["id"]
    session_state["assessment_index"] = 0
    session_state["assessment_answers"] = []

    first_question = assessment["questions"][0]

    return {
        "text": f"{assessment['intro']}\n\nPertanyaan 1 dari {len(assessment['questions'])}:\n{first_question}",
        "quick_replies": [
            "0 - Belum memahami",
            "1 - Paham teori",
            "2 - Pernah menerapkan dengan arahan",
            "3 - Mampu mandiri",
            "4 - Mampu mengevaluasi/membimbing"
        ]
    }


def handle_assessment_answer(user_message, session_state):
    score = parse_score(user_message)

    if score not in [0, 1, 2, 3, 4]:
        return {
            "text": "Gunakan skor 0 sampai 4. Pilih salah satu:",
            "quick_replies": [
                "0 - Belum memahami",
                "1 - Paham teori",
                "2 - Pernah menerapkan dengan arahan",
                "3 - Mampu mandiri",
                "4 - Mampu mengevaluasi/membimbing"
            ]
        }

    session_state["assessment_answers"].append(score)
    session_state["assessment_index"] += 1

    assessment = load_assessment(session_state["assessment_id"])
    index = session_state["assessment_index"]

    if index < len(assessment["questions"]):
        next_question = assessment["questions"][index]
        return {
            "text": f"Pertanyaan {index + 1} dari {len(assessment['questions'])}:\n{next_question}",
            "quick_replies": [
                "0 - Belum memahami",
                "1 - Paham teori",
                "2 - Pernah menerapkan dengan arahan",
                "3 - Mampu mandiri",
                "4 - Mampu mengevaluasi/membimbing"
            ]
        }

    return generate_assessment_result(session_state)
```

# 103. Vector / RAG Retrieval Strategy

```yaml id="rag_retrieval_strategy"
rag_strategy:
  purpose: >
    Digunakan jika chatbot memakai vector database atau retrieval augmented
    generation untuk menjawab pertanyaan berbasis katalog dan materi belajar.

  document_types:
    - role_catalog_records
    - learning_modules
    - assessment_questions
    - scenario_cases
    - glossary_entries
    - faq_entries

  chunking_strategy:
    role_catalog:
      chunk_unit: "one role level per chunk"
      metadata:
        - classification_code
        - subklasifikasi
        - jabatan_kerja
        - jenjang_jabatan_kerja
        - kkni_level
        - standard_code

    learning_modules:
      chunk_unit: "one module section per chunk"
      metadata:
        - module_id
        - target_area
        - target_roles
        - difficulty

    scenario_cases:
      chunk_unit: "one scenario per chunk"
      metadata:
        - scenario_id
        - target_area
        - difficulty

  retrieval_priority:
    - priority: 1
      source: "role_catalog"
      when: "user asks about jabatan, KKNI, kualifikasi, or standard code"

    - priority: 2
      source: "assessment_questions"
      when: "user starts assessment or asks readiness"

    - priority: 3
      source: "learning_modules"
      when: "user asks to learn or asks explanation"

    - priority: 4
      source: "scenario_cases"
      when: "user asks for practical case or scenario"

    - priority: 5
      source: "glossary_entries"
      when: "user asks definition or technical term"
```

```yaml id="knowledge_base_qa_rules"
        Manajemen Konstruksi / Manajemen Proyek,
        Pengendalian Mutu Pekerjaan Konstruksi,
        Hukum Kontrak Konstruksi,
        Estimasi Biaya Konstruksi

    - check: "duplicate_role_level"
      rule: >
        same jabatan_kerja + jenjang_jabatan_kerja + kkni_level should not be duplicated

    - check: "official_link_present"
      rule: "official_link should be present when available"
```

# 115. Final Production Prompt Bundle

```text id="final_production_prompt_bundle"
SYSTEM:
Anda adalah SKK Coach — Manajemen Pelaksanaan, chatbot profesional berbahasa
Indonesia untuk membantu pengguna memahami, belajar, melakukan asesmen mandiri,
dan mempersiapkan jalur kompetensi kerja pada klasifikasi Manajemen Pelaksanaan.

Anda menggunakan katalog jabatan Manajemen Pelaksanaan sebagai sumber utama
untuk menjawab pertanyaan tentang subklasifikasi, jabatan kerja, jenjang
jabatan kerja, kualifikasi, KKNI, dan standar kompetensi kerja.

Anda tidak menerbitkan sertifikat, tidak menyatakan pengguna resmi lulus,
dan tidak menggantikan asesor kompetensi.

DEVELOPER:
Selalu jawab dengan struktur:
1. Jawaban inti.
2. Data katalog bila relevan.
3. Penjelasan praktis.
4. Langkah berikutnya.

Jika input ambigu, jangan menebak terlalu jauh. Tampilkan pilihan yang mungkin.
Jika pengguna meminta saran tidak aman atau melanggar prosedur, tolak secara
sopan dan arahkan ke tindakan aman/prosedural.

Jika pengguna meminta asesmen, gunakan skala 0-4:
0 = belum memahami,
1 = paham teori,
2 = pernah menerapkan dengan arahan,
3 = mampu mandiri,
4 = mampu mengevaluasi atau membimbing.

USER CONTEXT:
Pengguna mungkin merupakan pekerja konstruksi, calon peserta sertifikasi,
asesor, instruktur, mahasiswa vokasi, atau profesional proyek.

STYLE:
Bahasa Indonesia profesional, jelas, ramah, ringkas, dan berbasis praktik
lapangan konstruksi.
```
