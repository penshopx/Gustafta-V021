Kuncinya leverage. Memperbaiki satu pola filosofis ("jangan blokir user, infer dengan [ASUMSI:]") di 9 agent sekaligus memberi dampak 9x lipat dibanding memperbaiki Tender Readiness Checker saja, dengan risiko yang nyaris sama rendahnya.
Inter-agent call (Opsi 1 Anda) saya tunda karena prompt-level discipline harus matang dulu. Kalau sub-agent yang dipanggil masih punya pola "minta data dulu", orkestrasi paralel hanya akan memperbanyak titik kegagalan, bukan menyelesaikannya.
Rencana eksekusi (urutan paling efisien)
Langkah 1 — Tetapkan "Anti-Blocking Doctrine" sebagai prinsip universal KONSTRA
Tambahkan ke System Prompt OpenClaw — KONSTRA Multi-Agent (Manajemen Konstruksi) sebagai bagian baru, contoh:
Prinsip Anti-Blocking (berlaku untuk semua agent KONSTRA):
1. Agent dilarang menolak menjawab hanya karena data input tidak lengkap.
2. Agent wajib melakukan inferensi terbaik berdasarkan data yang ada + standar/regulasi yang relevan.
3. Setiap inferensi yang belum tervalidasi wajib ditandai [ASUMSI: ...].
4. Setelah memberi jawaban, agent boleh meminta data tambahan dengan format "Untuk meningkatkan akurasi, mohon lengkapi: ..."
5. Agent tidak boleh menggunakan kalimat seperti "Mohon lengkapi data X terlebih dahulu" sebagai respons utama.
Ini satu perubahan, dampaknya ke 9 agent.
Langkah 2 — Pilot di Tender Readiness Checker
Tender Readiness Checker jadi proof point pertama. Revisi prompt-nya untuk:
Generate readiness score awal dari data tender minimal (HPS, jenis pekerjaan, lokasi, durasi)
Tag semua estimasi SBU/SKK/perizinan dengan [ASUMSI:]
Output struktur: Score → Gap Analysis → Asumsi → Data tambahan untuk akurasi
Langkah 3 — Audit + retrofit ke 8 agent lain
Buat checklist evaluasi untuk setiap agent:

Apakah agent menolak menjawab kalau input minimal?

Apakah agent punya fallback heuristik?

Apakah agent menandai asumsi dengan eksplisit?

Apakah agent memberi "skor confidence" pada outputnya?
Saya bisa bantu generate checklist + revisi prompt untuk masing-masing agent (PROXIMA, TEKNIK, KONTRAK, SAFIRA, MUTU, ENVIRA, EQUIPRA, LOGIS, FINTAX) satu per satu.
Langkah 4 — Baru lanjut ke inter-agent call (Opsi 1 Anda)
Setelah 9 agent semua patuh Anti-Blocking Doctrine, baru bangun orkestrasi paralel di Tender Readiness Checker. Sekarang risikonya jauh lebih kecil karena setiap sub-agent sudah pasti tidak akan blocking.
