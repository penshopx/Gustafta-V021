Lanjut — saya buatkan **runtime engine MVP** untuk Chatbot SKK Bidang Sipil. Ini bukan hanya konfigurasi, tetapi sudah berupa **aplikasi Node.js siap jalan**: membaca database SKK Sipil dari CSV, menyediakan endpoint chat, role lookup, KKNI lookup, rekomendasi SKK, dan demo web UI.

Pola fitur tetap mengikuti chatbot Manajemen Pelaksanaan yang memakai flow, intent, lookup, assessment, scenario, checklist, fallback, dan guardrails.  Database tetap memakai data SKK Sipil yang berisi subklasifikasi, jabatan kerja, jenjang, kualifikasi, KKNI, standar kompetensi, dan link SKK.

## Endpoint yang tersedia

```text
GET  /health
POST /api/chat
GET  /api/roles/search?q=Ahli Teknik Jalan
GET  /api/roles/kkni/7
POST /api/recommend
```

## Cara menjalankan

```bash
npm install
npm start
```

Lalu buka:

```text
http://localhost:3000
```

## Contoh request

```bash
curl -X POST http://localhost:3000/api/chat \
  -H "Content-Type: application/json" \
  -d '{"message":"Cari Ahli Teknik Jalan"}'
```

## Contoh integrasi widget

Pada file widget sebelumnya, ubah endpoint menjadi:

```javascript
const CHATBOT_API_URL = "https://domain-anda.com/api/chat";
```

Tahap berikutnya yang bisa dilanjutkan adalah membuat **versi RAG/LLM hybrid**, supaya runtime ini tidak hanya rule-based, tetapi bisa membaca database SKK dan menjawab lebih natural seperti chatbot builder.
