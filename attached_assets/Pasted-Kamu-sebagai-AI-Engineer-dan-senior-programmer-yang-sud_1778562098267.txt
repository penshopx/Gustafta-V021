 Yang sudah jadi
1 halaman hub + 6 database operasional + 1 custom agent yang sudah lulus tes ABD-7.
Struktur workspace
 Brain Project — hub utama (cara pakai, peran, doktrin ABD, standar acuan)
 Master Proyek — register semua proyek (nilai kontrak, status, sektor, deviasi otomatis)
 Laporan Harian (LHP) — input cuaca, mandays, progres, kendala + Flag Brain Project
 Register Risiko & Isu — skor risiko P×D otomatis + level ​
 NCR / Quality Control — non-conformance + RFI + tindakan koreksi/pencegahan
 K3 & Lingkungan — toolbox, JSA, PTW, insiden, CAPA + rujukan regulasi (UU 1/1970, PP 50/2012, Permenaker 5/2018, Permen PUPR 10/2021, UU 32/2009, dll.)
 Cost & Schedule Control (EVM) — SPI/CPI/EAC/VAC otomatis + status  On Track /  Watch /  Alert
Agent  Brain Project
Tiga topi: Konsultan • Pengawas (MK) • K3 & Lingkungan — diwajibkan cross-check antar topi.
Doktrin ABD (Anti-Blocking) sesuai pola internal Anda: best-effort answer, tag [ASUMSI: …], confidence score, output 7-bagian.
Aturan deteksi dini terkalibrasi (SPI/CPI <0,95 → Watch; <0,85 → Alert; NCR mayor >3 hari; near miss berulang; VO kumulatif mendekati 10%; dll.).
Akses edit sudah diberikan ke hub + 6 database; web search aktif untuk verifikasi regulasi/SNI.
Trigger: dipanggil via mention @Brain Project di halaman/komentar mana pun di tree.
Tes ABD-7
Saya jalankan skenario uji (proyek gedung 5 lantai Bekasi: deviasi −7%, beton kolom 24 MPa vs f'c 30, near miss harness, VO 8%). Agent merespon lengkap dalam format ABD-7 — ringkasan, asumsi eksplisit, analisis Q-C-T + K3-S, early warning, rekomendasi H/M/L dengan PIC, confidence 78%, dan pertanyaan tindak-lanjut. Lulus .
 Cara mulai

Mulai isi Laporan Harian setiap hari sore.
Mention @Brain Project di baris LHP atau di halaman proyek → minta: "Review LHP minggu ini", "Audit kepatuhan K3", "Hitung EAC & proyeksi delay", dll.
Saat ada temuan: catat di NCR, K3, atau Risiko & Isu — agent akan otomatis ikut analisis saat dipanggil.
