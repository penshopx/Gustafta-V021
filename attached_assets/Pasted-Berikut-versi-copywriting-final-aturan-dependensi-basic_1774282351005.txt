Berikut versi “copywriting final” + aturan dependensi (basic vs advanced), dengan istilah konsisten dan helper text maksimal 1 baris.

A) Konsistensi istilah (rekomendasi)

- Gunakan “Mode Agentic” (bukan “agentik”) karena ini istilah umum di produk AI.
- Gunakan “Otonomi” untuk autonomy, “Retensi konteks” untuk memory, “Bantuan proaktif” untuk proactivity.
- Hindari campur Bahasa Inggris kecuali nama preset (Balanced, Mentor, dll.) karena itu memang label produk.

C) Aturan dependensi & penguncian (rekomendasi implementasi)

1) behaviorPreset mengatur default turunan

- Saat user memilih preset (selain Custom), sistem mengisi nilai rekomendasi untuk:
    - autonomyLevel, responseDepth, proactiveAssistance, clarifyBeforeAnswer, uncertaintyHandling, showRiskWarnings, interactionStyle (dan yang lain jika perlu).
- UX: tampilkan tombol kecil “Reset ke Preset” di card Preset Perilaku (Advanced).
- Penguncian:
    - Opsi 1 (lebih jelas): tidak dikunci, tapi ada “Reset ke Preset”.
    - Opsi 2 (lebih ketat): sebagian field dikunci bila preset ≠ Custom, dan dibuka bila preset = Custom.
    - Rekomendasi saya: Opsi 1 (lebih fleksibel, lebih sedikit frustrasi).

2) Mode Lanjutan (Advanced) hanya mengatur visibilitas, bukan perilaku

- Advanced OFF: sembunyikan pengaturan detail, tapi nilai tetap tersimpan.
- Advanced ON: tampilkan semua.

3) proactiveHelpTypes muncul hanya jika proactiveAssistance ≠ Off

- Jika user set proactiveAssistance = Off, proactiveHelpTypes disembunyikan dan nilai tetap tersimpan (atau reset—pilih salah satu, saya sarankan tetap tersimpan).

4) actionBoundary mengontrol fitur “aksi”

- Jika “Boleh memanggil mini apps” tidak dipilih, maka UI/fitur mini apps tidak ditawarkan di percakapan.
- Jika “Boleh mengarahkan ke chatbot lain” tidak dipilih, jangan tampilkan rekomendasi chatbot spesialis.
- Jika “Boleh memberi CTA bisnis” tidak dipilih, jangan berikan CTA penjualan.

5) storeInteractionSignals hanya relevan jika ada “pembelajaran/analitik”

- Jika adaptiveLearningMode = Off, storeInteractionSignals boleh tetap ada (untuk analytics), atau disembunyikan supaya sederhana.
- Rekomendasi: tampilkan hanya di Advanced, tanpa dependensi ketat.

6) Mode Agentic OFF membatasi fitur multi-step

- Jika agenticMode = false:
    - multiStepReasoning dipaksa OFF atau diabaikan.
    - proactiveAssistance maksimal “Rendah” (opsional).
    - Tampilkan info: “Mode Agentic nonaktif, kemampuan perencanaan dibatasi.”

D) Default paket Basic vs Advanced (final)

- Basic (tampil): Preset Perilaku, Tingkat Otonomi, Mode Agentic, Kedalaman Jawaban, Format Output Utama, Minta Klarifikasi Dulu, Saat Tidak Yakin, Tampilkan Peringatan Risiko, Mendengarkan dengan Teliti, Retensi Konteks, Bantuan Proaktif, Gaya Interaksi
- Advanced (tambahan): Pemeriksaan Mandiri, Prioritas Konteks, Jenis Bantuan Proaktif, Empati Kontekstual, Batas Tindakan AI, Aturan Eskalasi, Saat di Luar Topik, Mode Pembelajaran Adaptif, Simpan Sinyal Interaksi, Prioritas Sumber Jawaban

- versi ringkas (1–2 baris per preset), atau
- versi detail (mapping preset → nilai untuk setiap field).
