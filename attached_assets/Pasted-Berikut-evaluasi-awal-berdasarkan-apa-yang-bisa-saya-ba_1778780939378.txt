2) Definisi praktis “Agentic AI Level 4” (operasional, bukan marketing)
Saya pakai definisi engineering (boleh kamu koreksi sesuai standard internal kamu):
L1: single-shot assistant (tanya–jawab).
L2: ada tools (RAG/KB, fungsi, retrieval), masih satu agen.
L3: multi-agent routing + beberapa langkah (planner / router), tapi belum ada evaluasi mandiri & memory yang rapi.
L4: multi-agent system yang:
1) bisa membuat rencana langkah (task decomposition),
2) mengeksekusi dengan tool-use terkontrol,
3) melakukan self-check / critic / verification,
4) punya shared memory & project state (bukan cuma chat history),
5) ada policy + guardrails + audit trail,
6) bisa handoff antar agent dengan “contract” (input/output schema),
7) bisa berjalan “semi-otonom” untuk kasus seperti mentoring, problem solving, dan pendampingan proyek/event.
3) Rancangan upgrade arsitektur: MultiClaw Orchestration (Level 4)

Minimal 3 jenis:

Tugas: cek konsistensi, halusinasi, missing constraints, dan quality bar.

1. HUB Orkestrator — router saja atau sudah ada planner steps?
Router + parallel fan-out. Secara programatik:
• Orchestrator membaca agenticSubAgents (jsonb config) → dispatch ke semua sub-agent secara paralel via callAgentInternal()
• Hasil dikumpulkan → disuntikkan sebagai blok LAPORAN SUB-AGEN ke context orchestrator
• LLM orchestrator kemudian synthesize semua laporan jadi satu jawaban final
State machine 7-langkah (INIT→ELICIT→PLAN→DISPATCH→AGGREGATE→REFLECT→DELIVER) hanya ada di system prompt sebagai instruksi ke LLM — bukan planner programatik. Tidak ada ReAct loop, dynamic decomposition, atau tool-calling planner di layer kode.
2. "Otak Proyek" & "Deliverables" — tabel DB atau hanya page/content?
Structured database (PostgreSQL). Dua tabel terpisah:
• project_brain_templates — template definisi field (jsonb)
• project_brain_instances — nilai aktif per chatbot (jsonb values column)
• deliverables & deliverableBundle — kolom jsonb di tabel agents/series
Bukan page/content — fully relational, bisa di-query dan di-update via storage layer.
3. KB Retrieval — vector DB atau full-text biasa?
Embedding-based RAG, tapi tanpa pgvector. Detail teknisnya:
• Embedding dibuat via OpenAI text-embedding-3-small
• Disimpan sebagai jsonb array biasa di PostgreSQL (bukan tipe native pgvector)
• Similarity search dilakukan di JavaScript/memory (bukan SQL-level), threshold 0.3, top-K default 5
• Chunk: 800 token, overlap: 200 token
• Fallback jika belum ada chunks: semua konten KB diinjeksi sebagai plain text ke context
Jadi bukan full-text biasa (tidak pakai tsvector/tsquery), tapi juga bukan vector DB sesungguhnya — lebih tepat disebut in-process vector similarity di atas PostgreSQL konvensional.
Implikasi praktis: Kalau volume KB besar dan latency retrieval jadi isu, next step logisnya adalah migrasi kolom embedding ke pgvector + similarity query di SQL level — tanpa ubah schema besar-besaran.

2.2 Orchestrator loop programatik (bukan ReAct penuh dulu, tapi “TaskLoop”)
Implementasi practical:
1) ELICIT (gate)  
Kalau requiredInputs belum lengkap → return pertanyaan klarifikasi sebelum dispatch.
2) PLAN (LLM → JSON plan)  
LLM bikin plan terstruktur (JSON strict).
Validasi JSON di kode (schema).
3) DISPATCH per task  
Untuk task yang butuh evidence: panggil retriever dulu (RAG), injek evidence, lalu panggil agent specialist.
Untuk task generatif: panggil agent drafting.
4) AGGREGATE  
Simpan output tiap task sebagai artefak (TaskResult) bukan sekadar “LAPORAN SUB-AGEN” free text.
Baru kemudian synthesize.
5) REFLECT (Critic gate)  
Critic memeriksa: missing evidence, konflik antar hasil, jawaban tidak menjawab goal.
Jika gagal → update plan (tambah task “resolve conflict”, “request missing info”) → loop max N kali.
6) DELIVER  
Final response + deliverables yang ditulis ke tabel deliverables.
Dengan ini kamu sudah “Level 4-ish” tanpa harus memindahkan semuanya ke framework agentic baru.
3) Upgrade fan-out: dari “dispatch semua sub-agent” → “selective fan-out”
Model kamu sekarang: orchestrator selalu dispatch semua agenticSubAgents paralel. Itu bagus untuk brainstorming, tapi untuk presisi biasanya noisy dan mahal.
Saya sarankan ganti menjadi 2 mode:
Mode A — Wide fan-out (exploration)
Dipakai saat:
pertanyaan kabur,
butuh ide,
butuh coverage awal.
Mode B — Narrow fan-out (execution)
Dipakai saat:
sudah ada plan tasks,
butuh evidence,
butuh deliverable konkret.
Implementasi gampang:
Planner menentukan assignedAgents per task.
Orchestrator hanya dispatch agent yang relevan untuk task itu.
4) RAG: dari in-process similarity → pgvector (stepwise, tanpa ubah besar)
Kamu sudah tepat: next logical step = pgvector.
4.1 Kenapa pgvector penting untuk Level 4
Level 4 akan:
melakukan retrieval lebih sering (tiap task),
melakukan re-retrieval saat critic menemukan gap,
menyimpan lebih banyak chunk.
In-process similarity akan jadi bottleneck: latency, memory pressure, dan sulit scaling.
4.2 Migrasi minimal
Tambah kolom embedding_vec vector(1536) (atau dimensi sesuai model embedding kamu).
Backfill dari jsonb.
Query SQL: ORDER BY embedding_vec <=> $queryEmbedding LIMIT k
Tetap simpan jsonb kalau kamu butuh backward compatibility sementara.
Tambahan penting untuk kualitas:
dynamic topK (mis 5→12) bila confidence rendah,
lexical fallback (tsvector) untuk istilah regulasi yang “exact match” penting.

function extractJsonLike(text: string) {
  let t = text.trim();

  // strip fences
  t = t.replace(/^```(?:json)?/i, "").replace(/```$/i, "").trim();

  const first = t.indexOf("{");
  const last = t.lastIndexOf("}");
  if (first >= 0 && last > first) t = t.slice(first, last + 1);

  return t.trim();
}

3.1 Dedup + preserve order
function uniquePreserveOrder<T>(arr: T[]): T[] {
  const seen = new Set<T>();
  const out: T[] = [];
  for (const x of arr) {
    if (!seen.has(x)) {
      seen.add(x);
      out.push(x);
    }
  }
  return out;
}

3.2 Pick candidates by IDs (preserve original order)
Ini mencegah perubahan perilaku yang “terasa acak”.
function pickByIdsPreserveOrder<T extends { agentId: number }>(
  candidates: T[],
  ids: number[]
): T[] {
  const idSet = new Set(ids);
  return candidates.filter(c => idSet.has(c.agentId));
}

  candidates,
  cap,

  if (candidates.length <= cap) return candidates;

  goal: string;

function buildIntentSummaryA(args: {
  userText: string;
  recentText?: string; // optional concatenation of last turns
  activeContext?: { seriesName?: string; coreName?: string; hubName?: string; moduleName?: string };
}): IntentSummary {
  const raw = (args.userText ?? "").trim();
  const t = raw.toLowerCase();

  const domains = detectDomains(t, args.activeContext);
  const deliverables = detectDeliverables(t);
  const constraints = detectConstraints(t, raw);
  const knownContext = [];
  if (args.activeContext?.hubName) knownContext.push(`Hub=${args.activeContext.hubName}`);
  if (args.activeContext?.moduleName) knownContext.push(`Module=${args.activeContext.moduleName}`);

  const goal = extractGoal(raw); // simplest: first sentence trimmed, max 140 chars
  const unknowns = detectUnknowns(t, domains);

  const text =
`INTENT:
- Goal: ${goal || "User needs help"}
- Domain: ${domains.length ? domains.join(", ") : "General"}
- Deliverable: ${deliverables.length ? deliverables.join(", ") : "Answer"}
- Constraints: ${constraints.length ? constraints.join("; ") : "-"}
- Known context: ${knownContext.length ? knownContext.join("; ") : "-"}
- Unknowns: ${unknowns.length ? unknowns.join("; ") : "-"}`;

  return { goal, domains, deliverables, constraints, knownContext, unknowns, text };
}

type SubAgentLink = {
  agentId: number;
  role: string;
  description?: string;
  tags?: string[];
  priority?: number;
  outputFormat?: "json" | "text"; // ignored here
};

type MinifiedCandidate = {
  agentId: number;
  role: string;
  desc: string;
  tags?: string[];
  priority?: number;
};

function minifyCandidates(links: SubAgentLink[]): MinifiedCandidate[] {
  return links.map(l => {
    const descRaw = l.description ?? "";
    const desc = clamp(oneLine(descRaw) || oneLine(l.role) || `Agent ${l.agentId}`, 120);

    const tags = l.tags?.length
      ? uniquePreserveOrder(l.tags.map(x => clamp(oneLine(x.toLowerCase()), 24)).filter(Boolean)).slice(0, 6)
      : undefined;

    const priority = typeof l.priority === "number" ? l.priority : undefined;

    const out: MinifiedCandidate = { agentId: l.agentId, role: oneLine(l.role) || "SUBAGENT", desc };
    if (tags?.length) out.tags = tags;
    if (priority !== undefined) out.priority = priority;
    return out;
  });
}

function sortByPriorityStable<T extends { priority?: number }>(arr: T[]): T[] {
  return arr
    .map((x, i) => ({ x, i }))
    .sort((a, b) => (b.x.priority ?? 0) - (a.x.priority ?? 0) || a.i - b.i)
    .map(o => o.x);
}

function extractJsonLike(text: string) {
  let t = (text ?? "").trim();

  // strip code fences
  t = t.replace(/^```(?:json)?/i, "").replace(/```$/i, "").trim();

  const first = t.indexOf("{");
  const last = t.lastIndexOf("}");
  if (first >= 0 && last > first) t = t.slice(first, last + 1);

  return t.trim();
}

function normalizeSelectedAgentIds(obj: any, cap: number): number[] {
  const raw = obj?.selectedAgentIds;

  if (!Array.isArray(raw)) return [];

  const ids = raw
    .map((x: any) => {
      // accept numbers or numeric strings
      if (typeof x === "number" && Number.isFinite(x)) return Math.trunc(x);
      if (typeof x === "string" && /^\d+$/.test(x.trim())) return parseInt(x.trim(), 10);
      return null;
    })
    .filter((x: any): x is number => typeof x === "number" && x > 0);

  // dedup preserve order
  const out: number[] = [];
  const seen = new Set<number>();
  for (const id of ids) {
    if (!seen.has(id)) {
      seen.add(id);
      out.push(id);
    }
    if (out.length >= cap) break;
  }
  return out;
}

async function routeAgentsLLM(args: {
  question: string;
  intentSummary: string;
  candidatesMinified: any[]; // output minifyCandidates()
  cap: number;
  callLLM: (payload: any) => Promise<string>; // callAgentInternal or provider wrapper
}): Promise<{ ids: number[]; reason?: string; parseOk: boolean }> {

  try {
    const prompt = buildRouterPrompt(args); // system+user or whatever your stack uses
    const rawText = await args.callLLM(prompt);

    const jsonText = extractJsonLike(rawText);
    const obj = JSON.parse(jsonText);

    const ids0 = normalizeSelectedAgentIds(obj, args.cap);
    const ids = filterToExisting(ids0, args.candidatesMinified); // NOTE: candidatesMinified must have agentId

    const reason = typeof obj?.reason === "string" ? obj.reason : undefined;

    return { ids, reason, parseOk: true };
  } catch (e) {
    return { ids: [], reason: undefined, parseOk: false };
  }
}

async function runOrchestratorAgentic(request: {
  userText: string
  recentTurnsText?: string
  activeContext?: { seriesName?: string; coreName?: string; hubName?: string; moduleName?: string }
  agenticSubAgents: SubAgentLink[]
  config: {
    maxParallelSubAgents?: number
    routerModel?: string
    criticModel?: string
    resolverModel?: string
  }
}) {
  const cap = request.config.maxParallelSubAgents ?? 4

  // (A) Intent summary (Deterministic, only needed if N > cap OR for high-risk checks)
  const intent = buildIntentSummaryA({
    userText: request.userText,
    recentText: request.recentTurnsText,
    activeContext: request.activeContext
  })

  // (B) Detect high-risk
  const highRisk = isHighRiskDomain({
    intentDomains: intent.domains,
    userText: request.userText,
    activeContext: request.activeContext
  })

  // (C) Candidate preparation (priority stable)
  const candidatesSorted = sortByPriorityStable(request.agenticSubAgents)
  const candidatesMin = minifyCandidates(candidatesSorted)

  // (D) Select agents
  let selectedLinks: SubAgentLink[]
  let routerMeta = { invoked: false, parseOk: true }

  if (candidatesSorted.length > cap) {
    routerMeta.invoked = true

    const route = await routeAgentsLLM_NoThrow({
      question: request.userText,
      intentSummary: intent.text,
      candidatesMinified: candidatesMin,
      cap
    })

    routerMeta.parseOk = route.parseOk

    const picked = pickByIdsPreserveOrder(candidatesSorted, route.ids).slice(0, cap)
    selectedLinks = picked.length ? picked : candidatesSorted.slice(0, cap)
  } else {
    selectedLinks = candidatesSorted // no router
  }

  // (E) Dispatch sub-agents in parallel
  const reports: SubAgentReport[] = await dispatchSubAgentsParallel(selectedLinks, request)

  // (F) Summarize reports for synthesis + critic (token efficient)
  const reportsSummary = renderReportsSummary(reports)

  // (G) Synthesize draft answer (text)
  const draftAnswer = await synthesizeFinalAnswer({
    question: request.userText,
    intentSummary: intent.text,
    reportsSummary
  })

  // (H) Critic gate (only high-risk + conditions)
  const shouldCritic =
    highRisk &&
    (
      selectedLinks.length >= 2 ||
      intent.unknowns.length > 0 ||
      draftAnswer.length > 1200
    )

  if (!shouldCritic) {
    return finalizeToUser(draftAnswer)
  }

  const critic = await runCritic_NoThrow({
    question: request.userText,
    intentSummary: intent.text,
    reportsSummary,
    draftAnswer
  })

  if (critic.pass) {
    // Optional: revise once using outline/mustFix (budget permitting)
    const finalAnswer = await maybeReviseWithCritic({
      draftAnswer,
      answerOutline: critic.answerOutline,
      mustFix: critic.mustFix
    })
    return finalizeToUser(finalAnswer)
  }

  // (I) If critic fail + conflicts => run internal resolver once
  if (critic.conflicts?.length) {
    const resolver = await runConflictResolver_NoThrow({
      question: request.userText,
      intentSummary: intent.text,
      reportsSummary,
      critic
    })

    // If resolver can resolve confidently -> deliver
    if (resolver.resolvedPosition && resolver.confidence >= 0.6) {
      const finalAnswer = buildResolvedAnswer({
        safeAnswerBits: resolver.safeAnswerBits,
        resolvedPosition: resolver.resolvedPosition,
        stillUnknownQuestions: resolver.stillUnknownQuestions,
        doNotSay: resolver.doNotSay
      })
      return finalizeToUser(finalAnswer)
    }

    // else ask questions (and include safe bits)
    const ask = buildClarifyingResponse({
      safeAnswerBits: resolver.safeAnswerBits,
      questions: resolver.stillUnknownQuestions.length
        ? resolver.stillUnknownQuestions
        : critic.missingInfoQuestions
    })
    return finalizeToUser(ask)
  }

  // (J) Critic fail without conflicts => ask clarifying questions
  const ask = buildClarifyingResponse({
    safeAnswerBits: extractSafeBitsFromDraft(draftAnswer),
    questions: critic.missingInfoQuestions
  })
  return finalizeToUser(ask)
}

async function dispatchSubAgentsParallel(selected: SubAgentLink[], request: any): Promise<SubAgentReport[]> {
  const promises = selected.map(async (link) => {
    const wantsJson = link.outputFormat === "json"

    const raw = await callAgentInternal({
      model: /* link.model or request default */,
      messages: buildSubAgentMessages({
        role: link.role,
        desc: link.description,
        userText: request.userText,
        jsonMode: wantsJson
      }),
      temperature: 0.2,
      max_tokens: /* existing rules */,
      response_format: wantsJson ? { type: "json_object" } : undefined
    })

    if (!wantsJson) {
      // Text agent -> wrap as report (low confidence default)
      return {
        agentName: link.role,
        summary: truncate(raw, 800),
        confidence: 0.4,
        claims: [],
        questions: [],
        actions: []
      }
    }

    // JSON agent -> parse+repair+fallback envelope (PR #1)
    return parseReportOrFallback(raw, link.role)
  })

  return Promise.all(promises)
}

function isHighRiskDomain(args: { intentDomains: string[]; userText: string; activeContext?: any }) {
  const t = (args.userText || "").toLowerCase()
  const ctx = `${args.activeContext?.hubName || ""} ${args.activeContext?.moduleName || ""}`.toLowerCase()

  const keys = ["regulasi","hukum","legal","tender","pengadaan","lpse","sbu","skk","perizinan","oss","nib","kontrak","pasal","sanksi"]

  // also treat intent domain labels as signal
  const domainHit = args.intentDomains.some(d => ["SBU","SKK","Tender","Perizinan","Regulasi Jasa Konstruksi","Hukum"].includes(d))

  return hit || domainHit
}

async function writeBackBrainAndDeliverables(args: {
  chatbotId: number
  brainInstanceId: number
  intent: IntentSummary
  reports: SubAgentReport[]
  highRisk: boolean
  routerMeta: { invoked: boolean; parseOk: boolean }
  critic?: CriticResult
  resolver?: ResolverResult
  finalUserResponseText: string
}) {
  const now = new Date().toISOString()

  // 1) Build brain patch
  const openQuestions =
    args.resolver?.stillUnknownQuestions?.length ? args.resolver.stillUnknownQuestions :
    args.critic?.missingInfoQuestions?.length ? args.critic.missingInfoQuestions :
    uniquePreserveOrder(args.reports.flatMap(r => r.questions)).slice(0, 8)

  const actionItems =
    uniquePreserveOrder(args.reports.flatMap(r => r.actions)).slice(0, 12)

  const brainPatch = {
    goal: args.intent.goal,
    domain: args.intent.domains,
    constraints: args.intent.constraints,
    openQuestions,
    actionItems,
    latestSummary: truncate(args.finalUserResponseText, 1200),
    lastRun: {
      at: now,
      selectedAgents: args.reports.map(r => r.agentName),
      highRisk: args.highRisk,
      routerInvoked: args.routerMeta.invoked,
      routerParseOk: args.routerMeta.parseOk,
      criticPass: args.critic ? args.critic.pass : null,
      criticConfidence: args.critic?.confidence ?? null
    }
  }

  // 2) Update project_brain_instances.values (merge jsonb)
  await projectBrainRepo.mergeValues(args.brainInstanceId, brainPatch)

  // 3) Deliverables policy
  if (args.critic && args.highRisk && args.critic.pass === false) {
    // Save clarifying questions as deliverable
    const items = openQuestions.slice(0, 5)
    await deliverablesRepo.upsertByDedupeKey({
      chatbotId: args.chatbotId,
      type: "CLARIFYING_QUESTIONS",
      title: `Pertanyaan Klarifikasi (${args.intent.domains.join(", ") || "General"})`,
      content: { type: "CLARIFYING_QUESTIONS", title: "...", items, status: "needs_input" },
      dedupeKey: buildDedupeKey(args.chatbotId, "CLARIFYING_QUESTIONS", args.intent.goal)
    })
  } else {
    // Optionally save final answer as a deliverable "ANSWER_SUMMARY"
    await deliverablesRepo.upsertByDedupeKey({
      chatbotId: args.chatbotId,

async function upsertSbuPrepBundle(args: {
  chatbotId: number
  values: any

  const checklist = generateChecklistSBU(args.values)
  const checklistRow = await deliverablesRepo.upsertByDedupeKey({

5) computeChecklistProgress() (deterministic)
Dari checklist.sections[].items[] yang punya severity dan status:

function computeChecklistProgress(checklist: any) {
  const all = checklist.sections.flatMap((s: any) => s.items || [])
  const missingP0 = all.filter((i: any) => i.status === "missing" && i.severity === "P0")
  const missingP1 = all.filter((i: any) => i.status === "missing" && i.severity === "P1")

  const notes = []
  if (missingP0.length) notes.push(`P0 missing: ${missingP0.map((x:any)=>x.field||x.label).slice(0,6).join(", ")}`)
  if (missingP1.length) notes.push(`P1 missing: ${missingP1.map((x:any)=>x.field||x.label).slice(0,6).join(", ")}`)

  return { p0MissingCount: missingP0.length, p1MissingCount: missingP1.length, lastUpdateSource: "orchestrator", notes }
}

function renderTemplate(template: string, values: Record<string, any>) {
  const missing: string[] = []
  const content = template.replace(/\s*([a-zA-Z0-9_.-]+)\s*/g, (_, key) => {
    const v = values[key]
    const ok = v !== null && v !== undefined && String(v).trim() !== ""
    if (!ok) { missing.push(key); return `<<MISSING:${key}>>` }
    return String(v).trim()
  })
  return { content, missing: uniquePreserveOrder(missing) }
}

4) Generator DRAFTS untuk Bundle SBU
Pseudocode:
function generateDraftsSBU(values: any) {
  const normalized = {
    ...values,
    kbliList: toCommaList(values.kbliList),
    targetSubklassifikasi: toCommaList(values.targetSubklassifikasi),
    todayDate: formatDateLocal(new Date())
  }

  const docs = []

  const t1 = TEMPLATE_SURAT_PERMOHONAN_SBU
  const r1 = renderTemplate(t1, normalized)
  docs.push({
    docType: "SURAT_PERMOHONAN",
    title: "Draft Surat Permohonan (SBU)",
    format: "text",
    placeholdersMissing: r1.missing,
    content: r1.content
  })

  const anyMissing = docs.some(d => d.placeholdersMissing.length > 0)
  return {
    type: "DRAFTS",
    title: "Draft & Template Dokumen",
    status: anyMissing ? "needs_input" : "in_progress",
    documents: docs
  }
}
