## LANJUTAN 9 — Data Import Schema & Knowledge Base Engine

Modul ini membuat chatbot punya struktur database yang rapi untuk menampung data dari tabel SBU, Permen PU No. 6 Tahun 2025, dan SK Dirjen Binakon No. 37/KPTS/DK/2025. Tujuannya agar bot tidak hanya menjawab dari teks bebas, tetapi bisa melakukan **lookup terstruktur**: kode SBU → KBLI → kualifikasi → TKK → peralatan → persyaratan → verifikasi OSS/SIJK. Permen PU No. 6 Tahun 2025 menampilkan standar per KBLI, termasuk ruang lingkup, subklasifikasi jasa konstruksi, kualifikasi, TKK, peralatan, persyaratan, verifikasi, dan kewajiban.  SK Dirjen Binakon No. 37/KPTS/DK/2025 memuat skema sertifikasi, kriteria penilaian, dokumen pendaftaran, surveilans, re-sertifikasi, dan formulir asesor. 

```yaml
# -----------------------------------------------------------------------------
# LANJUTAN 9 — DATA IMPORT SCHEMA & KNOWLEDGE BASE ENGINE
# -----------------------------------------------------------------------------

knowledge_base_engine:
  id: sbu_knowledge_base_engine_v1
  title: "Mesin Knowledge Base SBU"
  purpose:
    - menyimpan data SBU secara terstruktur
    - mendukung pencarian kode SBU, KBLI, klasifikasi, subklasifikasi, TKK, peralatan, dan persyaratan
    - mendukung validasi awal KBLI dengan SBU
    - mendukung checklist dokumen dan pra-verifikasi
    - mendukung pembaruan regulasi jika ada referensi baru
```

## 9. Data Import Pipeline

```yaml
data_import_pipeline:
  id: sbu_data_import_pipeline_v1
  stages:
    - stage: "extract"
      description: "Ambil data dari Excel/PDF/tabel regulasi."
      output:
        - raw_rows
        - raw_text_blocks

    - stage: "normalize"
      description: "Samakan format kode, KBLI, judul, klasifikasi, dan kualifikasi."
      rules:
        - "uppercase semua kode SBU"
        - "hapus spasi ganda"
        - "KBLI harus 5 digit"
        - "pisahkan kode SBU dan kode rancang bangun/terintegrasi bila ada"
        - "kelompokkan alat berdasarkan K, M/B, atau B"

    - stage: "map"
      description: "Petakan data ke tabel master."
      target_tables:
        - subklasifikasi_master
        - kbli_master
        - qualification_matrix
        - tkk_requirement_matrix
        - equipment_requirement_matrix
        - document_requirement_matrix
        - regulation_reference

    - stage: "validate"
      description: "Cek kualitas data."
      checks:
        - "setiap sbu_code punya kbli_code"
        - "setiap kbli_code punya minimal satu related_sbu_code"
        - "setiap sbu_code punya scope_summary"
        - "kode duplikat ditandai needs_review"
        - "KBLI ganda ditandai ambiguous"

    - stage: "publish"
      description: "Data siap dipakai chatbot."
      output:
        - searchable_catalog
        - lookup_index
        - keyword_index
        - compliance_index
```

## 11. Data Quality Rules

```yaml
data_quality_rules:
  sbu_code_format:
    rule: "Kode harus diawali BG, BS, IN, KK, KP, PA, PB, PL, atau kode lain yang valid sesuai regulasi."
    action_if_invalid: "mark_as_needs_review"

  kbli_format:
    rule: "KBLI harus 5 digit angka."
    action_if_invalid: "mark_as_invalid_kbli"

  multiple_sbu_for_same_kbli:
    rule: "Jika satu KBLI memiliki banyak subklasifikasi, tampilkan semua kandidat dan ajukan klarifikasi."
    action_if_detected: "mark_as_ambiguous_kbli"

  missing_scope:
    rule: "Jika ruang lingkup kosong, jangan jawab detail; minta admin melengkapi data."
    action_if_detected: "fallback_to_general_response"

  outdated_reference:
    rule: "Jika ada referensi lebih baru, gunakan referensi dengan priority tertinggi."
    action_if_detected: "show_regulatory_update_note"

  equipment_by_qualification:
    rule: "Jangan tampilkan daftar alat tanpa memperhatikan kualifikasi."
    action_if_detected: "ask_target_qualification"

  tkk_by_qualification:
    rule: "Jangan menilai jenjang TKK tanpa mengetahui kualifikasi target."
    action_if_detected: "ask_target_qualification"
```

## 15. Runtime Prompt Tambahan

```yaml
runtime_prompt_addition_data_engine:
  text: >
    Gunakan knowledge base terstruktur sebelum menjawab. Jika pengguna memberi
    kode SBU, lakukan exact lookup ke subklasifikasi_master. Jika pengguna memberi
    KBLI, lakukan lookup ke kbli_master dan tampilkan semua kandidat. Jika pengguna
    menanyakan TKK atau peralatan, minta kualifikasi target bila belum disebutkan.
    Jika data tidak lengkap, jangan mengarang; tampilkan status “data perlu review”
    dan minta admin melengkapi katalog. Jika ada perbedaan antara katalog lama dan
    referensi regulasi terbaru, tampilkan catatan validasi regulasi.
```

Dengan modul ini, chatbot sudah punya fondasi data yang kuat untuk dijadikan aplikasi yang bisa di-update dari admin panel:

**Upload katalog → normalisasi data → validasi KBLI/SBU → index pencarian → lookup detail → pra-verifikasi → rekomendasi dokumen/kepatuhan.**
