# 47. Search Scoring Logic

```yaml id="md04fu"
search_scoring:
  purpose: >
    Menentukan hasil paling relevan saat user mencari jabatan, standar,
    KKNI, atau bidang Mekanikal.

  scoring_rules:
    exact_match_jabatan_kerja: 100
    exact_match_jenjang_jabatan: 90
    exact_match_standar_kompetensi: 85
    exact_match_kkni: 80
    exact_match_subklasifikasi: 70
    keyword_match: 60
    partial_match_jabatan: 50
    partial_match_subklasifikasi: 40
    partial_match_standard: 35
    semantic_related_term: 25

  result_rules:
    - "Jika skor tertinggi >= 85 dan hanya satu hasil dominan, tampilkan detail jabatan."
    - "Jika ada 2–5 hasil relevan, tampilkan daftar pilihan."
    - "Jika hasil lebih dari 5, tampilkan 5 teratas dan minta user mempersempit pencarian."
    - "Jika skor rendah, gunakan fallback dengan saran kata kunci."
    - "Jika input ambigu seperti 'crane', 'las', 'alat berat', tampilkan kategori terkait."

  max_results_displayed: 5
```
