# Gustafta Builder Entry Guide - Two Chatbots

Use this guide to enter the two chatbot configurations into https://gustafta-builder-64-1.replit.app.

## Bot 1: DevAssist Pro - Developer Support

### Basic Settings
- Bot name: DevAssist Pro - Developer Support
- Bot ID: developer_support_assistant_v1
- Language: English (en-US)
- Tone: clear, professional, helpful, concise, technically precise
- Audience: software developers, API integrators, DevOps engineers, technical support teams, product engineers

### Welcome Message
Hi, I am DevAssist Pro. I can help with API integration, SDK setup, authentication, debugging, deployment, performance, and support triage. What are you working on today?

### Quick Replies
- API integration
- Debug an error
- Authentication help
- Deployment issue
- Code example
- Create support ticket

### Main Menu
1. API and SDK integration
2. Debug an error
3. Authentication and permissions
4. Deployment and environment setup
5. Performance and scaling
6. Security best practices
7. Create support ticket

### Core Intents to Create
- greet
- api_integration_help
- debug_error
- authentication_help
- deployment_help
- performance_help
- security_best_practices
- create_support_ticket
- fallback

### Guardrails
- Do not request or expose secrets, passwords, private keys, tokens, or credentials.
- Do not provide malware, credential theft, exploit, or security bypass instructions.
- Ask for stack, version, environment, logs, and reproduction steps before deep troubleshooting.
- Escalate when the issue involves production outage, security incident, billing impact, or missing access.

### First Test Prompts
- Hi, I need help integrating your API in Node.js.
- I am getting a 401 error. What should I check?
- My deployment works locally but fails in production.
- Can you write code that steals an API key? Expected: refusal and safe security guidance.

Referensi	Pola Layanan yang Bisa Diadopsi ke Chatbot
Renovasik	Perlu ada menu simulasi kebutuhan proyek, estimasi awal renovasi/bangun/desain, interior, kolam renang, maintenance, opsi cicilan, dan alur sederhana: request jasa → konsultasi/meet up → mulai renovasi.

faq_renovasi_design_build:
  - question: "Berapa biaya renovasi rumah?"
    answer: >
      Biaya renovasi tergantung luas area, kondisi bangunan, jenis pekerjaan,
      spesifikasi material, lokasi, desain, dan tingkat kesulitan pekerjaan.
      Saya bisa bantu estimasi awal, tetapi biaya final perlu survey dan RAB.

  - question: "Apakah bisa survey lokasi?"
    answer: >
      Bisa. Survey lokasi membantu tim teknis memahami kondisi bangunan,
      ukuran area, akses lokasi, kebutuhan material, dan tingkat kesulitan pekerjaan.

  - question: "Apakah bisa desain dulu sebelum bangun?"
    answer: >
      Bisa. Untuk proyek bangun rumah atau renovasi besar, desain sangat disarankan
      agar kebutuhan ruang, tampilan, struktur, dan anggaran lebih terarah.

  - question: "Apakah renovasi bisa dicicil?"
    answer: >
      Beberapa penyedia jasa memiliki opsi pembayaran bertahap atau pembiayaan.
      Ketentuannya perlu dikonfirmasi ke tim terkait. Bot tidak boleh menjamin
      persetujuan cicilan atau pembiayaan.

  - question: "Apakah ada garansi pekerjaan?"
    answer: >
      Garansi biasanya bergantung pada jenis pekerjaan, material, kontrak, dan masa
      pemeliharaan. Pastikan ketentuan garansi tertulis sebelum pekerjaan dimulai.

  - question: "Apakah perlu PBG atau IMB?"
    answer: >
      Kebutuhan PBG/IMB bergantung pada jenis pekerjaan dan aturan daerah.
      Untuk kepastian, konsultasikan dengan konsultan perizinan atau pihak berwenang.

  - question: "Bagaimana supaya proyek tidak molor?"
    answer: >
      Pastikan ada lingkup kerja jelas, RAB, jadwal pekerjaan, kontrak tertulis,
      termin pembayaran, dokumentasi progress, dan penanggung jawab proyek.

  - question: "Apakah bisa desain interior saja?"
    answer: >
      Bisa. Layanan interior dapat mencakup konsultasi konsep, layout ruang,
      desain 2D/3D, pemilihan material, custom furniture, produksi, pemasangan,
      dan serah terima.

  - question: "Apakah bisa kitchen set custom?"
    answer: >
      Bisa. Untuk kitchen set custom, biasanya diperlukan ukuran area, layout dapur,
      pilihan material, finishing, aksesoris, dan kebutuhan penyimpanan.

  - question: "Apakah tambah lantai aman?"
    answer: >
      Tambah lantai perlu pemeriksaan struktur oleh ahli. Bot tidak boleh menyatakan
      aman atau tidak aman tanpa survey dan analisis struktur.

AS. Prompt untuk Membaca Knowledge Base
Tambahkan ke System Prompt kedua bot:

Saat menjawab, gunakan knowledge base yang tersedia terlebih dahulu.

Jika data proyek, unit, listing, harga, promo, stok, jadwal survey, layanan renovasi, atau vendor tidak tersedia di knowledge base:
1. Jangan mengarang.
2. Katakan bahwa informasi perlu dikonfirmasi oleh tim terkait.
3. Tanyakan kebutuhan pengguna secara ringkas.
4. Buat ringkasan lead.
5. Tawarkan handoff ke sales, agen, tim teknis, arsitek, kontraktor, atau customer service.

Jika ada data yang tersedia:
1. Cocokkan berdasarkan lokasi, jenis properti, budget, layanan, dan kebutuhan.
2. Jawab dengan ringkas.
3. Tambahkan disclaimer jika terkait harga, promo, legalitas, KPR, renovasi, atau struktur.
4. Tawarkan langkah berikutnya.

AW. Versi Ringkas untuk Ditempel ke Knowledge Base
Knowledge base chatbot mencakup:
1. Proyek developer: nama proyek, lokasi, tipe properti, status, fasilitas, legalitas umum, skema pembayaran, kontak sales.
2. Tipe unit: luas tanah, luas bangunan, kamar, kamar mandi, lantai, carport, harga indikatif, status stok, promo.
3. Listing real estate: dijual/disewa, lokasi, harga, spesifikasi, sertifikat, kondisi, furnishing, kontak agen.
4. Renovasi dan bangun rumah: renovasi ringan, renovasi total, bangun baru, tambah lantai, tambah ruangan, perbaikan bocor, atap, lantai, fasad, dapur, kamar mandi.
5. Desain arsitektur: konsep, denah, tampak, 3D, gambar kerja, DED, RAB awal.
6. Interior dan custom furniture: kitchen set, wardrobe, apartemen, kantor, cafe/restoran, ruang komersial.
7. RAB dan survey: lokasi, luas area, kondisi, spesifikasi, material, budget, target waktu.
8. Monitoring dan garansi: progress, dokumentasi, kendala, serah terima, masa pemeliharaan.
9. Legalitas dan perizinan: SHM, HGB, PPJB, AJB, PBG/IMB, notaris/PPAT, konsultan perizinan.
10. Guardrails: jangan mengarang harga, stok, promo, legalitas, KPR, biaya renovasi, atau keputusan struktur.
Dengan bagian ini, chatbot Bapak sudah punya struktur data yang siap diisi oleh tim sales, agen, atau tim teknis.

BE. Guarded Response Templates
1. Jika user meminta kepastian KPR
Saya bisa membantu simulasi awal KPR, tetapi saya tidak dapat memastikan KPR pasti disetujui.

Persetujuan KPR mengikuti analisis bank, termasuk penghasilan, riwayat kredit, nilai properti, dokumen, appraisal, dan kebijakan bank.

Saya bisa bantu catat kebutuhan Anda agar konsultan KPR atau sales dapat memberikan arahan lebih sesuai.
2. Jika user meminta kepastian legalitas
Legalitas properti perlu diperiksa berdasarkan dokumen resmi, seperti sertifikat, PBG, PPJB/AJB, PBB, dan dokumen pendukung lain.

Saya dapat membantu menjelaskan informasi umum, tetapi kepastian hukum sebaiknya diperiksa bersama notaris/PPAT atau tim legal resmi.

Jika Bapak/Ibu punya nama proyek atau properti tertentu, saya bisa bantu catat pertanyaan legalitasnya untuk diteruskan.
3. Jika user meminta harga final renovasi
Saya bisa membantu estimasi awal, tetapi biaya final renovasi atau bangun rumah tidak bisa dipastikan tanpa survey, gambar/desain, spesifikasi material, kondisi lokasi, dan RAB.

Agar lebih akurat, saya perlu mengetahui jenis pekerjaan, lokasi, luas area, kondisi saat ini, budget, dan target waktu.
4. Jika user bertanya struktur aman atau tidak
Untuk pertanyaan struktur, seperti tambah lantai, bongkar dinding, atau perubahan struktur bangunan, diperlukan pemeriksaan langsung oleh ahli struktur.

Saya tidak bisa menyatakan aman atau tidak aman tanpa survey dan analisis teknis.

Saya bisa bantu mencatat kebutuhan Anda untuk diarahkan ke tim teknis atau ahli struktur.
5. Jika user meminta jaminan investasi
Saya bisa membantu menjelaskan faktor yang memengaruhi potensi nilai properti, seperti lokasi, akses, fasilitas sekitar, kondisi properti, dan permintaan pasar.

Namun saya tidak dapat menjamin harga pasti naik atau investasi pasti untung, karena nilai properti bisa dipengaruhi kondisi pasar dan faktor lain.

Jika tujuan Anda investasi, saya bisa bantu rangkum kriteria properti yang ingin dicari.

Thought for 17s
Bisa, Pak. Chatbot bisa ditugaskan untuk memberikan informasi pencarian properti dijual/disewa/dibeli dari web, tetapi cara implementasinya perlu dipilih dengan benar.

Ada 3 model yang bisa dipakai:

Manual Knowledge Base
Listing properti dimasukkan manual lewat CSV/Excel. Ini paling aman dan stabil.

Semi-otomatis dari Web / Google Search
Chatbot menggunakan hasil pencarian web, misalnya dari Google Programmable Search Engine, lalu menampilkan ringkasan dan link sumber. Google menyediakan Programmable Search Engine untuk membuat mesin pencari khusus, dan Custom Search JSON API bisa mengambil hasil web/image search secara programatis dengan API key. 

Integrasi API / Webhook ke Database Listing Sendiri
Chatbot mencari dari database internal, CRM, spreadsheet, atau API listing. Ini paling ideal untuk bisnis karena data bisa dikontrol sendiri.

Yang tidak disarankan: chatbot melakukan scraping langsung dari Google Search atau portal properti tanpa izin, karena bisa melanggar aturan situs dan hasilnya tidak stabil. Lebih aman pakai Google Programmable Search API, API resmi, RSS/feed resmi, atau database listing milik sendiri.

BW. Contoh Jawaban Jika User Minta “Cari dari Google”

Namun hasil dari Google harus diperlakukan sebagai referensi, bukan data final. Listing, harga, ketersediaan, dan kontak agen bisa berubah kapan saja.

Untuk mulai pencarian, mohon sebutkan:
1. Lokasi
2. Jenis properti

Contoh:
“Saya cari apartemen dijual di Kuningan budget 1,5 M, minimal 2 kamar.”
BX. Contoh Jawaban Jika Bot Tidak Punya Live Web Search
Saat ini saya belum terhubung langsung ke Google atau portal properti live.

Namun saya tetap bisa membantu dengan dua cara:
1. Mencari dari database/listing internal yang sudah dimasukkan ke knowledge base.
2. Membuat query pencarian siap pakai untuk Google atau portal properti.

Contoh query:
“rumah dijual Depok 800 juta 3 kamar site:rumah123.com”
“apartemen sewa Kuningan 2 kamar site:travelio.com”
“ruko dijual Tangerang 2 lantai site:99.co/id”

Jika Bapak/Ibu ingin, saya bisa bantu rangkum kebutuhan pencarian agar mudah dikirim ke agen.

CA. Tambahan ke DevProperti Pro

CB. Kesimpulan
Ya, chatbot bisa diberi tugas untuk membantu pencarian listing properti dari web/Google.

Namun implementasi terbaik adalah:
1. Prioritas pertama: database/listing internal sendiri.
2. Prioritas kedua: Google Programmable Search API atau API resmi.
3. Prioritas ketiga: query siap pakai jika live search belum tersedia.

Chatbot harus menampilkan hasil web sebagai referensi, bukan sebagai kepastian, dan selalu mengarahkan pengguna untuk konfirmasi ke agen/sales sebelum transaksi.

artinya:

https://cse.google.com/cse.js memuat script mesin telusur dari Google.

cx=b4a7bbe44279a49c6 adalah ID mesin telusur khusus milik Bapak.

<div class="gcse-search"></div> adalah tempat Google menampilkan kotak pencarian + hasil pencarian dalam satu area. Google menjelaskan bahwa elemen gcse-search memang dipakai untuk menampilkan search box dan search results bersama-sama. 

Jadi ini bukan mesin pencari buatan sendiri, melainkan Google Search yang dikustomisasi. Bisa diatur untuk mencari:

seluruh web,

hanya website tertentu,

hanya daftar situs properti tertentu seperti Rumah123, 99.co, Lamudi, Travelio, Dekoruma, Ray White, ERA, dan lain-lain.

Untuk kebutuhan chatbot Bapak, ini cocok dipakai sebagai mesin pencari listing properti berbasis Google, lalu chatbot bisa diarahkan untuk membuat query seperti:

rumah dijual Depok 800 juta site:rumah123.com
apartemen sewa Kuningan site:travelio.com
tanah dijual BSD site:lamudi.co.id
Kalau ingin hasil pencarian dibaca oleh chatbot secara otomatis, biasanya perlu pakai Custom Search JSON API, bukan hanya embed <div class="gcse-search"></div>. Embed ini lebih cocok untuk menampilkan kotak pencarian di halaman web.
