## 🟢 Sprint 1 — Foundation (4 agen)

**Target:** ORCHESTRATOR + ELIGIBILITY + RISKSCAN + WINPROB live & lulus 5 skenario Sprint 1.

### Pre-deploy

- [ ]  System Prompt final di-review PO + Prompt Engineer
- [ ]  KB minimum tersedia (KB-REG Tier-1 22 entries, KB-MARKET basic untuk WINPROB)
- [ ]  `callAgentInternal` matrix didefinisikan (ORCH ↔ 3 agen)
- [ ]  Test scenarios siap (5 skenario — lihat sub-page Sprint 1 Tests)

### Deploy ke Gustafta Builder

1. Login Gustafta → New Agent → Series TENDERA → L3 Agent
2. Paste System Prompt dari sub-page Orchestrator/MultiClaw
3. Set persona, greeting, conversation starters (sesuai Hub spec)
4. Set memory: long-term ON, session ON
5. Set tools: lookup KB-REG, lookup KB-MARKET (untuk WINPROB)
6. Save & Publish to staging

### Smoke Test

| Agen | Smoke Prompt | Expected |
| --- | --- | --- |
| ORCHESTRATOR | "Bagaimana cara mulai?" | Greeting + 7 conversation starters |
| ELIGIBILITY | Input minimum S1-01 | Verdict ELIGIBLE + ABD-7 lengkap |
| RISKSCAN | Paste klausul S1-03 | Verdict HIGH RISK + 3 temuan |
| WINPROB | Input minimum S1-04 | Score 60–80 + Top-3 Levers |

### Post-deploy

- [ ]  Jalankan 5 skenario S1-01 s.d. S1-05 penuh
- [ ]  Log semua percakapan
- [ ]  Checklist ABD-7 compliance per output
- [ ]  Sign-off PO

### Rollback

Jika ≥ 2 skenario fail dan tidak bisa fix dengan micro-patch <30 menit:

1. Unpublish agen di Gustafta
2. Re-deploy versi sebelumnya (atau placeholder "under maintenance")
3. Issue ticket di triage rubric (mirror [Issue Triage Rubric — Micro-Patch Library 9 Agen KONSTRA](https://www.notion.so/Issue-Triage-Rubric-Micro-Patch-Library-9-Agen-KONSTRA-e12f61e2ad3b4e0fa58b1944976fbd86?pvs=21))
4. Fix → re-deploy → re-test

## 🛡️ Risk & Mitigation

| Risk | Likelihood | Impact | Mitigation |
| --- | --- | --- | --- |
