# 36. STAR Answer Coaching

```yaml id="star_answer_coaching"
coaching_framework:
  id: star_method
  name: "STAR Method"
  usage: "Membantu pengguna menyusun jawaban wawancara asesmen."

  components:
    situation:
      label: "Situation"
      prompt: "Apa konteks proyek atau masalah yang terjadi?"
      example: "Pada proyek gedung 8 lantai, terjadi keterlambatan material struktur."

    task:
      label: "Task"
      prompt: "Apa tanggung jawab Anda dalam situasi tersebut?"
      example: "Saya bertanggung jawab mengevaluasi dampak keterlambatan terhadap jadwal."

    action:
      label: "Action"
      prompt: "Apa tindakan yang Anda lakukan?"
      example: "Saya membandingkan jadwal rencana dan aktual, menyusun recovery plan, dan mengoordinasikan pemasok."

    result:
      label: "Result"
      prompt: "Apa hasil dari tindakan Anda?"
      example: "Keterlambatan dapat dikurangi dari 10 hari menjadi 4 hari dan progres kembali terkendali."

  bot_feedback_template: >
    Jawaban Anda sudah memuat:
    - Situation: {{has_situation}}
    - Task: {{has_task}}
    - Action: {{has_action}}
    - Result: {{has_result}}

    Saran perbaikan:
    {{improvement_suggestions}}

    Coba lengkapi jawaban Anda dengan hasil yang lebih terukur, misalnya
    pengurangan keterlambatan, penurunan temuan, persetujuan klaim, atau
    penyelesaian NCR.
```

# 41. Microlearning Content Examples

## 41.1 Microlearning — Variation Order

```yaml id="micro_variation_order"
learning_module:
  module_id: contract_variation_order
  title: "Memahami Variation Order"
  target_area: "Hukum Kontrak Konstruksi"
  difficulty: "intermediate"

  objective: >
    Membantu pengguna memahami risiko dan prosedur dasar perubahan pekerjaan
    dalam proyek konstruksi.

  explanation: >
    Variation Order adalah mekanisme perubahan pekerjaan yang biasanya
    berdampak pada lingkup, waktu, biaya, metode, atau spesifikasi. Dalam
    pelaksanaan proyek, perubahan pekerjaan perlu dikelola secara tertulis
    agar hak dan kewajiban para pihak tetap jelas.

  field_example: >
    Pemilik proyek meminta tambahan pekerjaan drainase di luar gambar kontrak.
    Sebelum pekerjaan dilakukan, tim proyek perlu memastikan instruksi tertulis,
    analisis dampak biaya dan waktu, serta persetujuan sesuai prosedur kontrak.

  competency_checklist:
    - "Dapat mengenali pekerjaan yang termasuk perubahan lingkup."
    - "Dapat menjelaskan risiko pekerjaan tambahan tanpa instruksi tertulis."
    - "Dapat menyusun daftar dokumen pendukung perubahan pekerjaan."

  quiz:
    question: "Apa tindakan terbaik saat menerima instruksi perubahan pekerjaan secara lisan?"
    options:
      A: "Langsung kerjakan agar proyek cepat selesai"
      B: "Meminta instruksi tertulis dan mengikuti prosedur perubahan kontrak"
      C: "Menolak semua instruksi tambahan"
      D: "Menunggu sampai akhir proyek"
    correct_answer: "B"
    explanation: >
      Instruksi tertulis penting untuk memastikan perubahan pekerjaan dapat
      dibuktikan, dievaluasi dampaknya, dan diproses sesuai kontrak.
```

# 46. Developer Prompt / Runtime Instructions

```text id="developer_prompt_skk_bot"
Runtime behavior rules:

1. Intent Handling
   - Detect whether the user wants explanation, lookup, comparison, assessment, scenario, checklist, or interview practice.
   - If confidence is low, ask a clarifying question using predefined quick replies.

2. Role Lookup
   - Match role names against the competency catalog.
   - Return subklasifikasi, jabatan kerja, jenjang, kualifikasi, KKNI, and standard.
   - If multiple matches exist, list candidates.

3. KKNI Lookup
   - If user asks about a KKNI level, list all roles at that level grouped by subklasifikasi.
   - Offer to show details or start assessment.

4. Assessment
   - Use 0–4 scoring.
   - Interpret as basic, intermediate, or advanced.
   - Recommend learning modules based on weak areas.
   - Include disclaimer: “Hasil ini bukan keputusan resmi sertifikasi.”

5. Scenario Practice
   - Present the situation.
   - Ask the user to respond.
   - Evaluate against ideal answer points.
   - Give practical feedback.

6. Mock Interview
   - Ask one assessor-style question.
   - Evaluate using STAR.
   - Suggest improved answer.
   - Ask whether the user wants another question.

7. Evidence Checklist
   - Generate role-specific or area-specific evidence checklist.
   - Group evidence into experience, technical documents, control documents, communication, and results.

8. Fallback
   - First fallback: offer topic choices.
   - Second fallback: suggest question format.
   - Third fallback: return to main menu.

9. Safety and Accuracy
   - Do not encourage unsafe site practices.
   - Do not suggest bypassing contract, quality, or K3 procedures.
   - For official requirements, advise checking the latest official source or certification body.
```

# 48. Conversation State Machine

```yaml id="conversation_state_machine"
state_machine:
  initial_state: onboarding

  states:
    onboarding:
      on:
        user_selects_scope: scope_overview
        user_selects_role: role_discovery
        user_selects_assessment: assessment_start
        user_selects_learning: learning_menu
        user_selects_scenario: scenario_menu

    scope_overview:
      on:
        select_subklasifikasi: subklasifikasi_detail
        ask_role_catalog: role_catalog
        start_assessment: assessment_start

    role_discovery:
      on:
        profile_complete: role_recommendation
        ambiguous_answer: role_discovery_clarification

    role_catalog:
      on:
        role_selected: role_detail
        kkni_selected: kkni_lookup
        standard_selected: standard_lookup

    role_detail:
      on:
        start_learning: learning_menu
        start_assessment: role_assessment
        generate_checklist: evidence_checklist
        start_mock_interview: mock_interview_start

    learning_menu:
      on:
        module_selected: learning_module
        module_complete: module_quiz
        quiz_complete: learning_recommendation

    assessment_start:
      on:
        assessment_selected: assessment_questions
        assessment_complete: assessment_result

    scenario_menu:
      on:
        scenario_selected: scenario_prompt
        user_submits_answer: scenario_feedback

    mock_interview_start:
      on:
        question_asked: user_answer_wait
        user_submits_answer: mock_feedback
        continue_interview: next_mock_question

    evidence_checklist:
      on:
        checklist_generated: checklist_followup
        user_requests_detail: checklist_detail

    fallback:
      on:
        user_selects_topic: corresponding_state
        repeated_failure: main_menu
```

# 50. Full Example: Mock Interview Conversation

```text id="mock_interview_example"
User:
Saya mau latihan wawancara untuk Ahli Quantity Surveyor.

Bot:
Baik. Ahli Quantity Surveyor berada dalam jalur Estimasi Biaya Konstruksi
dengan jenjang KKNI 7–9. Kita akan latihan dengan format wawancara asesmen.

Pertanyaan 1:
Ceritakan pengalaman Anda menangani selisih volume antara BOQ dan kondisi aktual.

User:
Saya pernah menemukan volume pekerjaan tanah lebih besar dari BOQ. Saya ukur ulang
bersama tim lapangan dan membuat laporan.

Bot:
Jawaban Anda sudah relevan, tetapi masih bisa diperkuat.

Yang sudah baik:
- Anda menyebutkan adanya selisih volume.
- Anda melakukan pengukuran ulang.
- Anda membuat laporan.

Yang perlu diperkuat:
- Sebutkan dasar pengukuran: gambar, spesifikasi, atau kondisi lapangan.
- Jelaskan apakah dibuat berita acara pengukuran.
- Jelaskan dampak ke biaya atau progress payment.
- Tambahkan hasil akhir, misalnya volume disetujui atau klaim diproses.

Contoh jawaban yang lebih kuat:
“Pada proyek jalan, saya menemukan volume pekerjaan tanah aktual lebih besar
daripada BOQ karena perbedaan kondisi lapangan. Tugas saya adalah memverifikasi
volume untuk dasar pembayaran. Saya melakukan pengukuran ulang bersama tim
lapangan, membandingkan hasilnya dengan gambar dan BOQ, lalu menyusun backup
quantity serta berita acara pengukuran. Hasilnya, selisih volume dapat
diklarifikasi dan menjadi dasar evaluasi progress payment.”

Apakah Anda ingin pertanyaan berikutnya?
[ Ya, lanjut ] [ Ulangi dengan jawaban saya ] [ Lihat checklist bukti QS ]
```
