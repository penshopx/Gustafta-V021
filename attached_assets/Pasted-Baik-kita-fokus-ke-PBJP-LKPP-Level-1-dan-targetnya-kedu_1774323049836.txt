## A) Struktur Notion yang paling rapi untuk di-*Import* ke Gustafta (per chatbot)

Buat 1 folder utama:

- `KB — PBJP LKPP L1 — Pelatih & Portofolio`

Di dalamnya buat 3 subfolder (atau 3 halaman induk):

- `01 — Foundational`
- `02 — Operational`
- `03 — Case Memory`

**Aturan import ke Gustafta**

- 1 chatbot PBJP = import **folder ini saja**.
- Di Gustafta, layer:
    - Foundational → untuk definisi, prinsip, kerangka aturan
    - Operational → SOP belajar, SOP simulasi, template jawaban, rubrik, template portofolio
    - Case Memory → contoh kasus, contoh jawaban, kesalahan umum, contoh portofolio

## C) Contoh dokumen siap-tempel (PBJP L1) untuk Gustafta

### 1) SOP — Persiapan Uji Kompetensi PBJP LKPP Level 1 (Operational)

**Syarat**

- Menentukan target: PBJP LKPP Level 1 dan tanggal uji (jika sudah ada).
- Memiliki daftar materi/modul dan bank soal.
- Menyiapkan tempat menyimpan bukti/portofolio (folder drive/Notion).
- Menyiapkan dokumen kerja yang pernah dikerjakan (atau simulasi dokumen) untuk portofolio.

**Langkah-langkah**

1. Diagnostik (hari 1): kerjakan tryout 30 soal campuran, catat topik lemah.
2. Susun rencana H-14:
    - Hari 1–5: prinsip, etika, dan perencanaan pengadaan + 20–30 soal/hari.
    - Hari 6–10: pemilihan penyedia + 30–50 soal/hari + 1 studi kasus/hari.
    - Hari 11–13: kontrak & pelaksanaan + 30–50 soal/hari + latihan essay.
    - Hari 14: simulasi penuh + review error log.
3. Bangun “error log”: kelompokkan salah karena konsep, prosedur, atau ketelitian.
4. Bangun “cheat sheet” 1 halaman: prinsip, etika, urutan proses, dokumen kunci, jebakan umum.
5. Portofolio paralel (mulai hari 2):
    - kumpulkan bukti,
    - label,
    - mapping ke kompetensi,
    - buat narasi singkat per bukti.
6. Simulasi wawancara: latihan menjelaskan 3 bukti terbaik dengan struktur yang sama.

**Biaya**

- Umumnya terkait pendaftaran/pelatihan (jika ada). Biaya portofolio biasanya hanya untuk administrasi dan penyiapan dokumen.

**Waktu proses**

- 10–14 hari efektif untuk peningkatan skor + penguatan studi kasus.
- Portofolio bisa 3–7 hari tergantung ketersediaan bukti.

**Masalah umum**

- Masalah: skor tinggi di hafalan tapi rendah di kasus.
    - Solusi: pakai template analisis kasus “Masalah → Aturan → Langkah → Output dokumen → Risiko”.
- Masalah: portofolio banyak file tapi tidak meyakinkan.
    - Solusi: kurangi, pilih bukti yang paling relevan, lalu tambah narasi kontribusi dan konteks.
- Masalah: sering salah karena teliti/waktu.
    - Solusi: latihan simulasi timer + review pola kesalahan.

---

### 2) Template — Mapping Bukti Portofolio PBJP (Operational)

**Syarat**

- Minimal punya 5–12 bukti (dokumen, notulen, laporan, email resmi, screenshot sistem, dll.)

**Langkah-langkah**

1. Buat tabel mapping: Bukti → Tahap PBJP → Kompetensi → Peran → Ringkasan kontribusi → Tanggal → Validasi.
2. Untuk setiap bukti, tulis narasi 5–7 kalimat: konteks, tugas, aksi, hasil, risiko yang dikendalikan.
3. Tandai bukti “kuat” (autentik + relevan + terkini) dan “lemah” (kurang konteks, tidak jelas peran).

**Biaya**

- Tidak ada (kecuali biaya administrasi/cetak jika diperlukan).

**Waktu proses**

- 1–2 jam untuk membuat tabel awal.
- 15–30 menit per bukti untuk narasi yang rapi.

**Masalah umum**

- Masalah: bukti tidak menunjukkan peran pribadi.
    - Solusi: tambahkan bagian “Peran dan kontribusi” yang spesifik.
- Masalah: bukti tidak terkini.
    - Solusi: tambahkan bukti terbaru atau jelaskan relevansinya.

## D) Agar chatbot bisa “keduanya” tanpa bingung (mode percakapan)

Di Gustafta, sebaiknya Anda set “Orchestrator” atau instruksi bot seperti ini (konsep):

- Jika pengguna minta latihan/soal → masuk mode **Bank Soal**
- Jika pengguna minta kesiapan/asesmen → mode **Diagnostik**
- Jika pengguna minta bukti/portofolio → mode **Portofolio**
- Jika input kurang → bot wajib tanya 3 hal: target tanggal, pengalaman, fokus modul lemah
