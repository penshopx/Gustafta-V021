# 2. Test Evaluation Tracker (Database)

<aside>
📊

Database tracking 35-cell (5 bot Modul Tender × 7 test) akan ditambahkan di bawah halaman ini sebagai langkah berikutnya.

**Struktur:** Bot (select) | Test (select T1-T7) | Status (Pass/Fail/Partial/Not Started) | Tested Date | Output Snippet | Notes

**Goal:** 35/35 Pass sebelum lanjut Tahap 2.

</aside>

**Cara isi setelah database aktif:**

1. Setelah upgrade prompt v2.0 di Gustafta, jalankan T1-T7 untuk setiap dari 5 bot Modul Tender.
2. Untuk setiap test, buat 1 row baru.
3. Status: ✅ Pass | ❌ Fail | ⚠️ Partial | ⏳ Not Started.
4. Tulis snippet output bot di field Output Snippet untuk arsip.
5. Saat 35/35 Pass → freeze v2.0 sebagai baseline, lanjut ke Tahap 2 (§1).
6. Saat ada Fail → log gap, iterasi prompt v2.0.x.

# 3. Capability Inquiry — Pesan ke Tim Gustafta

<aside>
✉️

Copy template di bawah, kirim ke tim Gustafta Builder (via support email / forum / Discord). Tujuan: konfirmasi 6 kapabilitas teknis yang menentukan apakah federasi inter-bot bisa dibangun dalam Gustafta atau butuh n8n eksternal.

</aside>

```
Halo Tim Gustafta Builder,

Saya sedang membangun ~60 chatbot untuk ekosistem BUJK & sertifikasi konstruksi di akun saya (https://gustafta-builder-7-pixelstd.replit.app/). Untuk merancang arsitektur agentic multi-bot yang scalable, saya butuh konfirmasi 6 kapabilitas teknis berikut. Mohon dijawab apa adanya — "belum ada" sangat membantu agar saya bisa merencanakan workaround.

1) INTER-BOT MESSAGING API
   Apakah ada REST API yang bisa "send-message" dari kode eksternal (n8n / cURL) ke chatbot tertentu di akun saya, dan menerima respons-nya secara programmatic?
   - Endpoint URL?
   - Auth (API key / OAuth)?
   - Rate limit & quota?
   - Apakah Otak Proyek bot tersebut ikut ter-load dalam call tersebut?

2) OTAK PROYEK SHARED STATE
   Bisakah satu Otak Proyek (8-field konteks user) di-share antar chatbot dalam akun yang sama?
   - Atau setiap bot punya state terisolasi?
   - Adakah API untuk read/write Otak Proyek dari eksternal?

3) WEBHOOK OUT (event-driven)
   Bisakah Gustafta memanggil endpoint HTTP eksternal saat event tertentu?
   - Daftar event yang didukung (message_received, intent_detected, session_end, dll.)?
   - Payload format JSON?
   - Bisa filter event per bot tertentu?

4) FUNCTION / TOOL CALLING LLM
   Apakah dalam system prompt bot saya bisa mendefinisikan "tools" (HTTP API, database query, custom function) yang LLM bisa panggil otomatis (gaya OpenAI function calling)?
   - Format definisi tool?
   - Limit jumlah tool per bot?

5) VECTOR STORE / KB PERSISTEN
   Apakah ada KB persisten per user (lintas-sesi) yang bisa di-query via RAG?
   - Format upload (PDF, MD, JSON, URL crawl)?
   - Limit size / chunks?
   - Bisa diquery dari prompt secara eksplisit, atau otomatis di-inject?

6) SYSTEM PROMPT LIMITS & OVERRIDE
   - Max tokens system prompt per bot?
   - Apakah ada template / preset Gustafta yang OVERRIDE atau MERGE dengan system prompt user? (Saya curiga ada layer "meta-prompt" yang menambah instruksi default tertentu — perlu konfirmasi)
   - Apakah ada cara melihat efektif "final prompt" yang di-kirim ke LLM?

Konteks: Saya sudah mendiagnosis 2 anti-pattern di bot saya:
(a) "Directory pattern" — bot menyebut nama sub-agen lalu menyuruh user pindah manual.
(b) "Human-as-API / Interrogation Mode" — bot meminta user fetch data dari chatbot lain dan paste balik.

Kedua anti-pattern muncul karena prompt diset seakan-akan ada tool calling (variabel UPPERCASE seperti SKK_SUMMARY) padahal kapabilitasnya belum tentu tersedia. Konfirmasi 6 hal di atas akan menentukan: apakah saya scale dalam Gustafta langsung, atau perlu bangun layer eksternal (n8n + Supabase + LangFlow).

Terima kasih banyak atas bantuannya.

Salam,
Wuryanto Kusdjali
```

# 4. n8n External Orchestrator — Arsitektur

<aside>
🔌

**Skenario fallback** kalau Gustafta tidak/belum menyediakan inter-bot API. n8n berperan sebagai "agent fabric" eksternal yang mengkoordinasi 60 bot lewat webhook + HTTP. Self-host gratis di Replit/Railway/VPS.

</aside>

## 4.1 Diagram Arsitektur Tinggi-Level

```mermaid
flowchart TB
  User["👤 User"] -->|chat| GustaftaUI["Gustafta Builder UI"]
  GustaftaUI <-->|message| BotN["Bot N dari 60"]
  BotN -.->|webhook OUT| n8n["n8n Orchestrator<br>self-host di Replit"]
  n8n <-->|state R/W| Supabase[("Supabase<br>Otak Proyek shared")]
  n8n <-->|LLM call| OpenAI["OpenAI / Anthropic API"]
  n8n -.->|inter-bot API| OtherBots["Bot 1, 2, ... 59"]
  n8n --> Notify["Notify<br>Email / WA / Slack"]
```

## 4.2 Komponen & Cost

| Komponen | Peran | Stack | Cost |
| --- | --- | --- | --- |
| n8n | Workflow orchestrator (DAG) | n8n self-host (Replit/Railway/VPS) | Free / $5-10/bln VPS |
| Supabase | Shared memory + vector store + auth | Supabase free tier (500MB DB, 1GB storage) | Free |
| LLM Provider | Reasoning + planning | OpenAI / Anthropic / OpenRouter | $0.01-0.10 per request |
| Notification | Push ke user lintas-channel | n8n built-in nodes (Email/WA/Slack/Telegram) | Free |

## 4.3 Flow Tipikal — Tender × SBU × SKK Federation

```mermaid
sequenceDiagram
  participant U as User
  participant T as Modul Tender
  participant W as n8n
  participant S as SBU Bot

  T->>W: Webhook intent tender_check

  par Parallel fetch
    W->>S: GET sbu summary
  and
    W->>K: GET skk summary
  end
  S-->>W: SBU summary JSON
  K-->>W: SKK summary JSON
  W->>DB: Write merged context
  W-->>T: Aggregated response
  T-->>U: Final answer terstruktur
```

## 4.4 Prerequisite Teknis (dari §3 capability inquiry)

- ✅ **Webhook OUT** (Q3) → wajib agar n8n bisa di-trigger.
- ✅ **Inter-bot API** (Q1) → wajib agar n8n bisa panggil bot lain.
- ✅ **Function calling** (Q4) → opsional (tapi sangat membantu).

Kalau **3 prerequisite tidak terpenuhi**, n8n tetap berguna untuk:

- **Notification layer** (user paste output → n8n → email/WA)
- **Background analytics** (log + dashboard)
- **Knowledge base updater** (cron sync regulasi terbaru ke Supabase)

Tapi BUKAN untuk real-time inter-bot federation. Dalam skenario itu, opsi alternatif: pindah ke platform lain (Botpress, Voiceflow, Flowise, LangFlow, atau custom build).

## 4.5 Roadmap Implementasi n8n (kalau go-ahead)

1. **Week 1** — Setup n8n di Replit, connect Supabase, buat skema Otak Proyek.
2. **Week 2** — Build 1 webhook flow: Modul Tender → fetch SBU + SKK → aggregate.
3. **Week 3** — Test end-to-end dengan 1 user nyata.
4. **Week 4** — Replikasi ke 5 hub utama (SBU/SKK/LSP/Compliance/Operasional).
5. **Month 2-3** — Scale ke 60 bot, observability dashboard.

# 5. Konsolidasi 60 Bot — Cetak Biru Final

<aside>
🧹

**Tujuan:** Kurangi duplikat & tumpang tindih, perjelas hierarki, simpan ~52 bot dari 60. Hemat maintenance, kurangi user confusion, percepat onboarding.

</aside>

## 5.1 Duplikat untuk Dimerge

| Klaster | Bot Sekarang | Rekomendasi | Hasil |
| --- | --- | --- | --- |
| **SBU Variants** | SBU Pekerjaan Konstruksi Terintegrasi · SBU All-in-One · SBU Pekerjaan Konstruksi · SBU Konsultan · SBU Jasa Penunjang Tenaga Listrik · SBU Migas/EBT/Pertambangan | Merge 3 SBU konstruksi umum → 1 bot universal. Pertahankan 2 specialist (Konsultan, Migas/Tenaga Listrik) karena regulasi spesifik. | 6 → 3 bot |
| **SMAP Variants** | SMAP & PANCEK · SMAP ISO 37001 · SMAP Nasional | Merge ke 1 bot "SMAP Compliance Hub" dengan 3 persona internal (ISO 37001, PANCEK, Nasional) | 3 → 1 bot |
| **Personel Manajerial 9 Role (×2)** | 2 versi duplikat | Dedupe ke 1 | 2 → 1 bot |

**Penghematan total:** 11 bot → 5 bot (hemat **6 bot**). Plus optimasi minor di klaster lain bisa hemat 2 bot lagi.
