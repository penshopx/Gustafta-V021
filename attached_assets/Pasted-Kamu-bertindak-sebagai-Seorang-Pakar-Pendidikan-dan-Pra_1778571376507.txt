 B — KB Skeleton (5 database aktif + 4 two-way relation)
Bank Soal — 16 properti (Bloom C1–C6, IRT logit, status review, tag UTBK/ANBK/OSN) + 4 view (Tabel, Per Mapel, Per Kesulitan, Siap Pakai)
Profil Siswa — 17 properti (jenjang, target, VARK, XP, streak, rank, mood, consent) + 3 view (termasuk Leaderboard XP)

Miskonsepsi Log — 13 properti (kategori 6 jenis, keparahan, frekuensi, remediasi) + 3 view
Rubrik Penilaian — 16 properti (8 jenis asesmen, skala, 6 kriteria + deskriptor) + 3 view
Relasi antar database (otomatis dua arah):
Sesi Belajar ↔ Profil Siswa (Siswa / Sesi Belajar)
Miskonsepsi Log ↔ Profil Siswa (Siswa Terdeteksi / Miskonsepsi Tercatat)
Bank Soal ↔ Miskonsepsi Log (Miskonsepsi Terkait / Soal Pemicu)
Sesi Belajar ↔ Rubrik Penilaian (Rubrik Dipakai / Sesi Pemakaian)
 C — Detail Mode
Gamification Engine: tabel XP resmi (11 baris + cap harian 1.000), kurva level Lv 1–100 dengan 5 rank, 7 kategori badge × 4 kelangkaan, sistem quest harian/mingguan/seasonal, mekanik Boss Battle lengkap, wellness gate anti-adiksi.
Adaptive Tryout: 5 mode tryout, algoritma IRT 1-PL (Rasch) dengan pseudocode, anti-cheating, post-mortem 5-bagian (skor + heatmap + time-per-question + mistake taxonomy + 3-step roadmap).

Tulis 5 specialist prompt lanjutan — DrillAgent, TryoutAgent, LiteracyAgent, GamificationAgent, MentorAgent (Sprint S2–S5).
Buat halaman Smoke Test ABD — 10 skenario per agent + rubrik acceptance, paralel dengan pola audit KONSTRA Anda.
Bangun Parent Dashboard — database laporan mingguan + view dashboard chart untuk orang tua.
