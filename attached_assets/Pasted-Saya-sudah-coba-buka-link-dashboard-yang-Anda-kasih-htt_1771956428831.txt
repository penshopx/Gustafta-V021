## Yang perlu kita pastikan di Gustafta Builder (cek di UI Anda)

Untuk saran “Optional Advanced Insight Layer” (yang generik), builder cukup punya 4 kemampuan ini:

1. **State/Variable**
   Ada tempat menyimpan output langkah sebelumnya, misal:

* `status`
* `risk_level`
* `score`
* `summary_json`

2. **Conditional branching / rules**
   Bisa bikin aturan seperti:

* jika `risk_level in ["Tinggi","Kritis"]` → tampilkan opsi lanjutan
* jika `score >= 70` → jalur A, else jalur B

3. **Button / quick reply / action** (opsional tapi ideal)
   Ada elemen “klik untuk lanjut” seperti:

* “Generate Advanced Summary”
* “Lihat Analisis Detail”

Kalau tidak ada tombol pun, masih bisa pakai trigger berbasis teks:
user ketik “buat executive summary”.

4. **Multi-step flow / subflow**
   Bisa memanggil langkah lanjutan (subflow) yang mengambil data dari hasil langkah pertama.

Kalau 4 hal ini ada, maka fitur yang saya sarankan **tetap relevan** dan **tetap generik**, karena di level platform namanya bukan “Executive Compliance Summary”, tapi:

> **Optional Advanced Insight Layer** (label & isi bisa dikustom tiap chatbot)

## Kalau ternyata builder belum punya tombol/conditional, masih bisa jalan?

Masih. Versi paling kompatibel (paling generik) adalah:

* Output basic selalu keluar
* Di akhir ada kalimat: “Ketik **RINGKASAN LANJUTAN** untuk analisis mendalam”
* Lalu keyword itu memicu subflow “Advanced Insight”

Ini **tidak menambah fitur khusus**; cuma memanfaatkan pattern percakapan umum.
