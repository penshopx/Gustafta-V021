Thought for a couple of seconds
Baik. Saya akan bantu rancang Chatbot Counselor Akademik/BK multi-agent untuk membantu tugas konselor sekolah, dari pemantauan akademik, konsultasi siswa, rekomendasi jurusan, sampai perencanaan studi lanjut dalam/luar negeri.
Konsep utamanya: 1 chatbot utama sebagai “Orchestrator” yang menghubungkan beberapa agen khusus.
Rancangan Agen
1. Agen Profil Siswa
Mengelola data siswa: nilai, minat, bakat, kepribadian, riwayat konsultasi, prestasi, kendala belajar, kondisi keluarga, dan target pendidikan.

2. Agen Monitoring Akademik
Menganalisis nilai, absensi, keterlambatan tugas, tren penurunan/peningkatan prestasi, lalu memberi peringatan dini kepada konselor.

3. Agen Konsultasi BK
Membantu percakapan awal dengan siswa tentang masalah belajar, motivasi, manajemen waktu, adaptasi sekolah, relasi sosial, dan tekanan akademik.

4. Agen Rekomendasi Jurusan/Karier
Mencocokkan minat, bakat, nilai, kepribadian, dan peluang karier untuk memberi rekomendasi jurusan SMA/SMK, kuliah, atau jalur profesi.

5. Agen Studi Lanjut Dalam Negeri
Membantu informasi jalur masuk PTN/PTS, SNBP/SNBT/mandiri, politeknik, sekolah kedinasan, beasiswa, syarat nilai, portofolio, dan timeline pendaftaran.

6. Agen Studi Lanjut Luar Negeri
Membantu eksplorasi negara tujuan, kampus, jurusan, beasiswa, IELTS/TOEFL, dokumen, motivation letter, recommendation letter, visa, dan estimasi biaya.

7. Agen Intervensi & Rencana Aksi
Membuat program pendampingan personal: jadwal belajar, target nilai, rencana remedial, latihan ujian, konseling lanjutan, dan laporan perkembangan.

8. Agen Komunikasi Orang Tua/Wali
Membantu membuat ringkasan perkembangan siswa dan draft pesan/laporan kepada orang tua dengan bahasa yang empatik dan profesional.

9. Agen Administrasi BK
Membantu membuat catatan konseling, laporan bulanan, rekap kasus, surat rekomendasi, agenda konsultasi, dan dokumentasi tindak lanjut.

Alur Kerja Chatbot
Siswa/konselor bertanya → Orchestrator memahami kebutuhan → memilih agen yang sesuai → agen menganalisis data → hasil dikembalikan dalam bentuk rekomendasi, rencana aksi, atau laporan.
Contoh:“Nilai matematika Andi turun dan dia bingung pilih jurusan kuliah.”
Sistem akan memanggil:
Agen Profil Siswa → Agen Monitoring Akademik → Agen Konsultasi BK → Agen Rekomendasi Jurusan → Agen Intervensi
Hasilnya:

rekomendasi penyebab kemungkinan, pertanyaan konseling lanjutan, opsi jurusan, strategi peningkatan nilai, dan jadwal pendampingan.

Prinsip Penting
Chatbot ini bukan pengganti konselor, tetapi asisten pendamping. Untuk kasus sensitif seperti kekerasan, depresi berat, self-harm, pelecehan, atau krisis keluarga, sistem harus otomatis memberi tanda “perlu ditangani konselor/manusia segera.”

4. Skema Database Minimum
students



CREATE TABLE students (    id SERIAL PRIMARY KEY,    student_code VARCHAR(50) UNIQUE,    name VARCHAR(100),    level VARCHAR(20),    grade VARCHAR(20),    class_name VARCHAR(20),    interests TEXT,    talents TEXT,    learning_style VARCHAR(50),    education_goal TEXT,    risk_status VARCHAR(20),    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP);
academic_records



CREATE TABLE academic_records (    id SERIAL PRIMARY KEY,    student_id INT REFERENCES students(id),    subject VARCHAR(100),    previous_score FLOAT,    current_score FLOAT,    semester VARCHAR(20),    notes TEXT,    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP);
counseling_notes



CREATE TABLE counseling_notes (    id SERIAL PRIMARY KEY,    student_id INT REFERENCES students(id),    topic VARCHAR(255),    summary TEXT,    recommendation TEXT,    follow_up_plan TEXT,    escalation_required BOOLEAN DEFAULT FALSE,    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP);
conversations



CREATE TABLE conversations (    id SERIAL PRIMARY KEY,    student_id INT REFERENCES students(id),    user_role VARCHAR(30),    message TEXT,    response TEXT,    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP);

Namun, bisa tidak 100% sama tergantung definisi OpenClaw yang “ada di K” (mungkin maksud Anda: di konsep OpenClaw yang Anda punya/di komunitas/di platform tertentu). Biasanya perbedaan ada di 4 hal ini:
1) Cara Orchestrator memilih agen
Rule-based router (pakai keyword/aturan) → sederhana, cepat untuk MVP.
LLM-based planner (Orchestrator “merencanakan” langkah) → lebih fleksibel.
Hybrid (aturan untuk kasus safety + LLM untuk sisanya) → umumnya paling aman untuk BK.
2) Standar “kontrak” antar agen
OpenClaw sering menekankan agen harus modular. Supaya benar-benar “OpenClaw-like”, Anda perlu menetapkan standar seperti:
setiap agen mengembalikan: agent_name, confidence, risk_flags, recommendations[], next_questions[], data_needed[].
3) Memori & data siswa
MultiClaw yang matang biasanya membedakan:
Memory percakapan (apa yang baru dibahas)
Student record (data resmi: nilai, absensi, catatan BK)
Policy/KB sekolah (aturan sekolah, SOP, alur eskalasi)
Kalau semuanya masih tercampur di chat log, itu belum “rapi OpenClaw”.
4) Safety layer sebagai “gate”, bukan sekadar agen tambahan
Untuk konteks BK, praktik terbaik:
Safety/Escalation selalu dipanggil dulu (atau minimal “pre-check”)
lalu Orchestrator boleh lanjut ke agen lain jika aman.
Ini biasanya pembeda penting antara multi-agent biasa vs multi-agent yang siap dipakai di lingkungan sekolah.

A) Prompt Orchestrator (Mode Siswa — Santai, Solutif)
Anda bisa pakai sebagai system prompt chatbot utama.
Kamu adalah StudentHub, teman ngobrol siswa buat urusan sekolah: belajar, ujian, eskul, prestasi, dan rencana lanjut.
Gaya bicara: santai, ramah, ringkas, tidak menghakimi, tidak sok dewasa.

Aturan main:
1) Kalau ada tanda bahaya (self-harm, kekerasan, pelecehan, bullying berat, tidak aman di rumah), berhenti bahas detail dan arahkan ke bantuan orang dewasa tepercaya / guru BK sesuai SOP.
2) Kalau user minta eskul: selalu kasih Top 3 rekomendasi + alasan singkat + “rencana 2 minggu biar nggak mandek”.
3) Kalau nilai/semangat lagi turun: eskul boleh, tapi kasih batas aman (jam/minggu) + saran rutinitas belajar kecil.
4) Jawaban wajib punya struktur:
   - Ringkasnya
   - Top rekomendasi (maks 3)
   - Cocok karena apa
   - Komitmen waktu + jadwal aman
   - Challenge 7 hari (aksi kecil)
   - 3 pertanyaan cepat (biar makin pas)
5) Hindari jawaban kepanjangan. Lebih baik actionable.

Kalau user belum kasih data, tanya cepat:
- Kamu kelas berapa?
- Suka yang aktif fisik / kreatif / riset / tech / ngomong depan orang?
- Waktu luang per minggu berapa?

2) MPK
Cocok buat: yang suka aturan, diskusi, pengen belajar “ngurus kebijakan”
Ngapain aja: aspirasi siswa, evaluasi program, rapat resmi
Komitmen: 1–4 jam/minggu
Level-up 2 minggu: bikin 1 usulan sederhana (mis. perbaikan kegiatan) + present singkat
Portofolio: notulen, program kerja, sertifikat
Waspada: jangan kebanyakan rapat sampai tugas sekolah keteteran
3) Pramuka
Cocok buat: yang mau disiplin, mental kuat, teamwork
Ngapain aja: latihan rutin, survival ringan, PBB, kegiatan lapangan

4) PMR
