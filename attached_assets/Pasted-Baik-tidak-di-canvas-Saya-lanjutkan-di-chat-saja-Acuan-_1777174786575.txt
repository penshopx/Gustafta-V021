Baik, tidak di canvas. Saya lanjutkan **di chat saja**.

Acuan struktur tetap mengikuti file terlampir: konfigurasi builder-style dengan **greeting flow, main menu, intents, sample utterances, response templates, quick replies, conversation branches, scoring logic, fallback, scenario practice, dan session summary**. # 35. Handler Priority

```yaml
handler_priority:
  order:
    - safety_guardrail
    - exact_standard_lookup
    - exact_sector_lookup
    - exact_role_or_scope_lookup
    - area_disambiguation
    - learning_assessment_evidence_scenario
    - glossary_lookup
    - fallback

  rationale:
    safety_guardrail: >
      Selalu diproses paling awal agar bot tidak memberi saran yang
      membahayakan keselamatan, mutu, kontrak, atau kepatuhan.
    exact_standard_lookup: >
      Jika pengguna menyebut kode standar/skema, sistem harus mengutamakan
      lookup katalog.
    area_disambiguation: >
      Jika pengguna hanya menyebut "migas", "EBT", "tambang", "K3", atau
      "PLTS", bot harus meminta pemilihan area lebih spesifik.
```

```ts
async function handleMessage(input) {
  const safety = await safetyGuardrail(input.message);
  if (safety.triggered) {
    return renderSafetyResponse(safety);
  }

  const normalized = normalizeText(input.message);
  const intent = detectIntent(normalized);
  const entities = extractEntities(normalized);

  const route = resolveRoute(intent, entities);

  const response = await executeRoute(route, {
    message: input.message,
    entities,
    session: input.session_state
  });

  await updateSession(response.session_update);
  await trackAnalytics(response.analytics_events);

  return response;
}
```
