## 1. Tujuan & Value Proposition

Generator Document dirancang untuk **memangkas waktu drafting dokumen SMAP dari berminggu-minggu menjadi hitungan jam**, dengan kualitas yang konsisten dengan SNI ISO 37001:2016 dan konteks BUJK Indonesia.

| **Aspek** | **Manual (Tanpa Generator)** | **Dengan Generator Document** |
| --- | --- | --- |
| Waktu drafting Kebijakan SMAP | 2–4 minggu (konsultan/internal) | 30–60 menit (review + signing) |
| Konsistensi antar dokumen | Variatif, sering kontradiktif | Cross-reference otomatis |
| Penyesuaian konteks BUJK | Sering generic / template kaku | Adaptive ke ukuran, klasifikasi, lingkup |
| Versioning & traceability | Manual filename `_v3_final_FINAL` | Otomatis dengan changelog |
| Update saat regulasi berubah | Refactor manual seluruh dokumen | Re-generate dengan delta diff |

### 3.2 Sumber Onboarding

Data dapat di-onboarding melalui **5 saluran**:

1. **Dialog interaktif** dengan chatbot (default, paling user-friendly).
2. **Form upload** template `.xlsx` profil BUJK (download → fill → upload).
3. **Integrasi sistem** existing (HRIS, ERP, Vendor Master) via API/CSV import.
4. **Dokumen unggah** (Akta Pendirian, SK Direksi, struktur organisasi PDF) → AI ekstraksi.
5. **Bulk import** untuk grup BUJK multi-entitas (1 file → N profil).

### 3.3 Validasi & Quality Check

Sebelum profil di-save, chatbot menjalankan validasi:

- **Mandatory field** lengkap (kolom ✅).
- **Konsistensi internal** (mis. jumlah personel ≥ jumlah Direksi + Manajer Lini).
- **Format check** (NPWP 15 digit, NIB 13 digit, alamat punya kota+provinsi).
- **Cross-check eksternal** (opsional): NIB ke OSS, NPWP ke DJP, daftar hitam LKPP.
- **Confidence score** profil (0–100%) — di bawah 70% chatbot akan minta klarifikasi tambahan.

## 4. Mekanisme Generation

### 4.1 Arsitektur Pipeline

```mermaid
flowchart TB
	U[👤 User Request] --> O[SMAP-ORCHESTRATOR]
	O --> P[AGENT-POLICY]
	P --> Profile[(Profil BUJK)]
	P --> KB[(Knowledge Base<br>Klausul + Template)]
	P --> Risk[(Risk Register)]
	P --> Gap[(Gap Checklist)]
	Profile --> Engine[Template Engine<br>+ Variable Substitution]
	KB --> Engine
	Risk --> Engine
	Gap --> Engine
	Engine --> Draft[Draft v0.1]
	Draft --> QC{Quality Gate<br>self-check}
	QC -->|Lulus| Preview[Preview ke User]
	QC -->|Gagal| Engine
	Preview --> Edit{User edit?}
	Edit -->|Ya| Iter[Iterasi v0.2 / v0.3]
	Edit -->|Tidak| Approve[Routing ke Approver]
	Approve --> Sign[Approver e-sign]
	Sign --> Pub[(Daftar Induk Dokumen<br>v1.0 Effective)]
```

### 4.2 Template Engine

Dokumen di-generate dari template **terstruktur** dengan slot:

- **Static text** — kalimat baku ISO 37001 yang wajib ada (tidak boleh di-edit AI).
- **Variable** — `nama_pt`, `nama_dirut`, `tanggal_efektif`, `kualifikasi`.
- **Conditional block** — paragraf yang muncul/hilang berdasar profil (mis. klausul agen hanya untuk BUJK yang menggunakan agen).
- **AI-generated section** — bagian narasi adaptif (mis. "Konteks Organisasi" disesuaikan tipe BUJK).
- **Reference section** — auto-pull dari database (mis. daftar proses bisnis dari Risk Register).

### 4.3 Contoh Mapping Variabel — Kebijakan SMAP (POL-001)

| **Slot di Template** | **Sumber Data** | **Contoh Output** |
| --- | --- | --- |
| `nama_pt` | Profil BUJK · Identitas | PT Karya Bangun Persada |
| `lingkup_smap` | Profil · Klasifikasi + Wilayah | seluruh aktivitas konstruksi gedung & sipil di wilayah Jawa, Sumatera, dan Sulawesi |
| `komitmen_dirut` | AI-generated dari template komitmen + nama Dirut | Saya, Ir. Hartono Wijaya, sebagai Direktur Utama PT Karya Bangun Persada, menyatakan komitmen penuh ... |
| `daftar_kategori_risiko` | Auto-pull [](https://www.notion.so/57ce809991524a329edba82689127992?pvs=21) · proses aktif | P1, P2, P5, P6, P9 (5 kategori dari 10 diaktifkan) |
| `nama_fkap` | Profil · FKAP | Bapak/Ibu [TBD] (akan diangkat melalui SK terlampir) |
| `tanggal_efektif` | Auto = tanggal approve + 7 hari kerja | 15 Mei 2026 |
| `nomor_dokumen` | Auto-increment dari Daftar Induk Dokumen | POL-SMAP-001/V1.0/2026 |

## 6. Quality Control — Self-Check

Sebelum draft di-show ke user, AGENT-POLICY menjalankan **8 quality gate**:

| **Gate** | **Cek** | **Tools** |
| --- | --- | --- |
| Q1 Completeness | Semua section wajib ada (per checklist klausul) | Klausul checker |
| Q2 Clause Coverage | Setiap requirement ISO 37001 ter-cover ≥ 1 paragraf | Cross-ref dengan [](https://www.notion.so/ec4bf89def7e40c5bb92d9f9d1602e97?pvs=21) |
| Q3 Consistency | Nomor dokumen, nama PT, tanggal konsisten lintas dokumen | Variable diff |
| Q4 Regulatory Fit | Klausul tidak bertentangan dengan UU Tipikor, Permen PUPR, PanCEK | KB regulasi |
| Q5 Variable Filled | Tidak ada `...` tersisa di output final | Regex scanner |
| Q6 Tone & Bahasa | Bahasa Indonesia formal, tense konsisten | Style linter |
| Q7 Cross-Reference | Setiap rujukan dokumen lain valid (POL → SOP → Form) | Document master check |
| Q8 Risk Alignment | Kontrol di SOP cover Top-5 risiko di Risk Register | Risk-control mapping |

Dokumen hanya di-publish ke **Daftar Induk Dokumen** jika lulus **8/8 gates**. Jika ada warning, ditunjukkan ke user dengan rekomendasi perbaikan.

## 7. Approval & Versioning Workflow

```mermaid
stateDiagram-v2
	[*] --> Draft
	Draft --> ReviewFKAP : Submit for review
	ReviewFKAP --> Draft : Comment / revisi
	ReviewFKAP --> ApproveDireksi : FKAP approve
	ApproveDireksi --> Draft : Direksi minta revisi
	ApproveDireksi --> Effective : Direksi approve + e-sign
	Effective --> Periodic : Periodic review
	Periodic --> Effective : No change
	Periodic --> Draft : Major change → versi baru
	Effective --> Obsolete : Superseded oleh versi baru
	Obsolete --> [*]
```

### 7.1 Versioning Rule

- **v0.x** — draft, belum di-approve.
- **v1.0** — approved, effective.
- **v1.1, v1.2** — minor revision (typo, klarifikasi); tidak butuh approve ulang Direksi.
- **v2.0** — major revision (perubahan substansi, regulasi); butuh re-approve Direksi.
- Maksimal 1 versi effective per dokumen pada satu waktu.

### 7.2 Audit Trail

Setiap draft & revisi otomatis log:

- Tanggal & waktu generate
- User yang request
- AI model & versi prompt
- Profil BUJK snapshot saat generate
- Diff dengan versi sebelumnya
- Approval chain (siapa approve, kapan, ID e-sign)

## 10. Keamanan & Privacy

<aside>
🔒

**Prinsip:** Data internal perusahaan yang di-onboarding adalah **rahasia** dan tidak boleh keluar dari workspace BUJK.

</aside>

- **Tenancy isolation** — profil BUJK terisolasi per workspace.
- **Encryption at rest & in transit** untuk semua field profil.
- **PII handling** — nama Dirut, FKAP, manajer di-hash di log; full-name hanya di output dokumen.
- **No training contamination** — data internal **tidak digunakan** untuk training model.
- **RBAC** — hanya user dengan role `policy.draft` boleh generate; role `policy.approve` untuk approve.
- **Audit log** retensi minimal 5 tahun (sesuai best practice ISO 37001 + UU Tipikor).
- **Data deletion** — request hapus profil & semua draft via FKAP, executed dalam 14 hari kerja.
