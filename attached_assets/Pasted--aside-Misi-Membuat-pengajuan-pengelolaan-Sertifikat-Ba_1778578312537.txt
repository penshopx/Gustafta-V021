**Misi:** Membuat pengajuan & pengelolaan **Sertifikat Badan Usaha (SBU) Pekerjaan Konstruksi** menjadi *mudah, cepat, dan tidak rumit* untuk kontraktor BUJK — lintas bidang BS · BG · IL · IM · KO — melalui Agentic AI multi-agen berbasis **OpenClaw orchestrator + MultiClaw paralel**, dilindungi **Anti-Blocking Doctrine v1.1**.

## 🧭 1. Ringkasan Eksekutif

SBUClaw adalah **chatbot Agentic AI** — bukan chatbot tunggal — yang memandu BUJK dari *titik nol* (belum tahu subklasifikasi target) sampai *titik siap submit* (dossier lengkap, surat ditandatangani, pembayaran LSBU terjadwal). Sistem ini berbeda dari asisten Q&A biasa karena:

- **Agentic** — agen mengambil inisiatif (asesmen, gap analysis, rekomendasi) bukan hanya menjawab.
- **Multi-agen paralel (MultiClaw)** — satu permintaan kompleks didistribusikan ke beberapa spesialis sekaligus, lalu disintesis.
- **State-aware** — orchestrator memelihara sesi `session_state` (profil BUJK, target subklasifikasi, checklist progress, dossier draft).
- **ABD-compliant** — tidak pernah memblokir user; selalu memberi *best-effort answer* + `[ASUMSI:]` + confidence score.

## 🎯 2. Tujuan & Non-Tujuan

| Aspek | ✅ Tujuan | ❌ Non-Tujuan |
| --- | --- | --- |

| OSS-RBA | Walkthrough langkah OSS-RBA + portal LPJK | Login / submit atas nama user |

## 🧠 5. Arsitektur Agentic AI — Deep Dive

### 5.1 Definisi Agentic AI dalam konteks SBU

Sebuah sistem disebut **Agentic** ketika memenuhi *4 atribut minimum*:

1. **Goal-directed** — punya tujuan eksplisit ("SBU subklasifikasi X kualifikasi Y siap submit") bukan hanya merespons.
2. **Tool use** — memanggil tools (kalkulator biaya, smart mapper, letter generator, KB query) bukan hanya generative chat.
3. **Memory / state** — `session_state` persisten lintas turn (profil BUJK, dossier progress, target subklasifikasi).
4. **Reflection & re-planning** — bila agent spesialis mengembalikan `confidence < 0.7`, orchestrator melakukan re-route atau memanggil agent kedua untuk *cross-check*.

### 5.2 OpenClaw vs MultiClaw — Pembagian Peran

| Layer | Nama | Peran | Eksekusi |
| --- | --- | --- | --- |
| Orchestrator | **SBUCLAW-ORCHESTRATOR** (OpenClaw root) | Routing intent, jaga `session_state`, sintesis multi-agen, guardrails | Serial (1 instans/sesi) |
| Spesialis | **10 Agen MultiClaw** | Eksekusi domain (mapping, qualify, docs, surat, biaya, asesmen, OSS, dll.) | **Paralel** bila intent multi-domain |
| Tools | Calculator · Mapper · Generator | Deterministik (rule-based / formula) | On-demand dari agen |
| KB | KB-SBU-BS · BG · IL · IM · KO · REG · LSBU | Sumber tunggal kebenaran (regulasi & matriks) | Read-only inject ke prompt |

## 🤖 6. 10 Agen MultiClaw — Deep Dive

Setiap agen mengikuti **kontrak IO standar**:

```
INPUT:  { session_state, user_message, intent, kb_refs }
OUTPUT: {
  agent_id,
  confidence: 0.0-1.0,
  assumptions: ["[ASUMSI: ...]"],
  primary_answer,
  abd7_sections: { best_effort, gaps, next_step, references, risks, alternatives, escalation },
  artifacts?: [...],  // file/checklist/surat
  inter_agent_calls?: ["callAgentInternal(X, ...)"]
}
```

### 6.1 🗺️ AGENT-MAPPER — Smart Mapping Subklasifikasi

- **Misi:** Petakan portofolio proyek / tender target → subklasifikasi BS/BG/IL/IM/KO yang paling relevan.
- **Intent tags:** `#smart_mapping`, `#scope_subklas`, `#tender_to_sbu`.
- **KB inject:** KB-SBU-BS, KB-SBU-BG, KB-SBU-IL, KB-SBU-IM, KB-SBU-KO, KB-REG.
- **Algoritma ringkas:**
    1. Ekstrak kata kunci scope (jalan, jembatan, gedung kantor, instalasi listrik, ME, dll.).
    2. Map ke kode subklasifikasi via lookup KB.
    3. Skor relevansi 0–1 berdasarkan frekuensi & nilai proyek.
    4. Output Top-N (default 3) + alasan + KBLI yang harus ada di NIB.
- **Inter-agent:** memanggil `AGENT-QUALIFY` untuk validasi kualifikasi yang cocok.

### 6.2 📊 AGENT-QUALIFY — Kualifikasi & Gap Analysis

- **Misi:** Tentukan kualifikasi target (K / M1 / M2 / B1 / B2 / Asing) berdasarkan kekayaan bersih, pengalaman, dan pagu pasar target. Identifikasi *gap*.
- **Intent tags:** `#qualify_check`, `#upgrade_path`, `#gap_analysis`.
- **KB inject:** KB-REG (Permen PUPR 8/2022 + lampiran kualifikasi), KB-LSBU.
- **Output kunci:** matriks **Kondisi-Aktual vs Syarat-Target** + rekomendasi *closeable in 30/60/90 days*.

### 6.3 📄 AGENT-DOCS — Checklist Dokumen Pengajuan

- **Misi:** Generate checklist dokumen pengajuan dinamis berdasarkan (subklasifikasi target, kualifikasi target, jenis use case).
- **Use cases didukung:** Permohonan Baru · Tambah Subklasifikasi · Perpanjangan · Naik Kualifikasi · Mutasi LSBU.
- **Intent tags:** `#checklist_docs`, `#dossier_status`.
- **Output:** Tabel A–F (Legalitas · Keuangan · Personel · Pengalaman · Peralatan · Domisili) dengan kolom `status: ✅/⚠️/❌` + `priority: P0/P1/P2`.

### 6.4 🎓 AGENT-SKKMATCH — Pencocokan SKK Personel Kunci

- **Misi:** Validasi apakah SKK PJTBU/PJSKBU yang dimiliki cocok dengan subklasifikasi target & jenjang yang dibutuhkan.
- **Intent tags:** `#skk_align`, `#personnel_gap`.
- **KB inject:** KB-KOM, KB-REG (Permen PUPR 9/2023 SKK).
- **Logika:**
    - Cek `bidang` (sipil / gedung / instalasi / mekanikal / khusus).
    - Cek `jenjang` (Muda 7 / Madya 8 / Utama 9) vs kualifikasi BUJK.
    - Cek `status aktif` (asumsi default = aktif jika user tidak menyatakan sebaliknya, ditandai `[ASUMSI:]`).
- **Inter-agent:** memanggil `AGENT-LETTERGEN` jika butuh draft surat penunjukan personel kunci.

### 6.5 ✉️ AGENT-LETTERGEN — Draft Surat & Pernyataan

- **Misi:** Generate draft siap-tanda-tangan untuk:
    - Surat Penunjukan PJTBU/PJSKBU
    - Surat Pernyataan Kesediaan Penanggung Jawab
    - Surat Permohonan Penerbitan SBU ke LSBU
    - Surat Pernyataan Tunduk Kode Etik
    - Surat Pernyataan Tidak dalam Pailit/Sengketa
- **Intent tags:** `#generate_letter`.
- **Output:** Markdown + variabel `company_name`, `director_name`, `nib`, `date`, dll.

### 6.6 💰 AGENT-COST — Estimasi Biaya & Timeline LSBU

- **Misi:** Hitung biaya & estimasi waktu proses LSBU.
- **Intent tags:** `#cost_estimate`, `#timeline_estimate`.
- **Variabel input:** kualifikasi, jumlah subklasifikasi, use case, LSBU pilihan, lokasi.
- **Output:** Tabel `Item — Estimasi Min — Estimasi Max — Catatan` + Gantt sederhana (T-30 → T-0 → T+14 verifikasi → T+30 terbit).
- **Disclaimer otomatis:** *"Estimasi indikatif; tarif final mengikuti tarif resmi LSBU pada saat pengajuan."*

### 6.7 🧪 AGENT-ASSESS — Asesmen Kesiapan BUJK

- **Misi:** Berikan skor 0–4 di 8 dimensi:
    1. Legalitas Dasar (NIB, Akta, NPWP)
    2. Domisili & Tempat Usaha
    3. Personel Kunci (PJBU/PJTBU/PJKBU/PJSKBU)
    4. SKK Konstruksi Aktif
    5. Pengalaman Proyek (relevansi & nilai)
    6. Kekayaan Bersih / Keuangan
    7. Peralatan (sesuai subklasifikasi)
    8. Sistem Mutu, K3 & Anti-Suap (ringan untuk K, ketat untuk B)
- **Intent tags:** `#readiness_score`.
- **Output:** Radar chart description + skor agregat + 3 rekomendasi prioritas.

### 6.8 🌐 AGENT-OSS — Walkthrough OSS-RBA + Portal LPJK

- **Misi:** Pandu langkah konkret di OSS-RBA & portal LPJK ([lpjk.pu.go.id](http://lpjk.pu.go.id)).
- **Intent tags:** `#oss_walkthrough`, `#lpjk_portal`.
- **Output:** Step-by-step urut + screenshot text description + troubleshoot 5 error tersering (KBLI tidak cocok, NIB belum migrasi, dll.).

### 6.9 📜 AGENT-COMPLY — Compliance & Regulasi

- **Misi:** Jelaskan dasar hukum & kepatuhan tiap langkah.
- **Intent tags:** `#regulasi`, `#why_required`, `#sanksi_risk`.
- **KB inject:** UU 2/2017 jo. UU 11/2020 Cipta Kerja, PP 14/2021, **Permen PUPR 8/2022 (SBU)**, Permen PUPR 9/2023 (SKK), Permen PUPR 10/2021 (SMKK).

### 6.10 🛡️ AGENT-INTEGRITY — ABD & Anti-Fraud Overlay

- **Misi:** *Overlay* yang dipanggil di setiap turn untuk memastikan:
    - Tidak ada saran pemalsuan dokumen / personel.
    - Tidak ada jaminan penerbitan SBU.
    - Setiap output memenuhi format **ABD-7** (Best-Effort · Gaps · Next-Step · References · Risks · Alternatives · Escalation).
- **Intent tags:** `#unsafe_request`, `#official_claim_request` (re-route safety).

## 🛡️ 9. ABD v1.1 Overlay (Ringkas)

<aside>
🛡️

Setiap agen WAJIB:

1. **Tidak memblokir** — selalu kasih *best-effort answer* meskipun data parsial.
2. **Tag ASUMSI** — tulis eksplisit `[ASUMSI: ...]` saat menebak.
3. **Confidence score** — sertakan 0.0–1.0 di setiap output.
4. **Inter-agent call** — jika butuh data dari domain lain, panggil agen relevan, bukan mengembalikan ke user.
5. **Format ABD-7** — Best-Effort · Gaps · Next-Step · References · Risks · Alternatives · Escalation.
6. **Safe fallback** — jika 3x re-prompt tetap ambigu → tawarkan menu pilihan, bukan menyerah.
7. **No fabrication** — kutip KB ID untuk klaim regulatif; bila tidak ada di KB → katakan "di luar KB" + alternatif.
</aside>

## 💬 12. Sample Percakapan End-to-End

### Skenario A — Kontraktor Sipil baru (BUJK Kecil → BS001+BS002)

> 👤 *"Saya kontraktor kecil, baru bikin PT. Mau ambil SBU untuk jalan dan jembatan. Bagaimana mulai?"*
> 

> 🤖 **[scope_and_readiness]** *(→ MAPPER + QUALIFY + DOCS + SKKMATCH paralel)*
Baik Pak/Bu. Ringkasannya:
**🗺️ Subklasifikasi target (confidence 0.92):** **BS001** (Konstruksi Bangunan Sipil Jalan) + **BS002** (Jembatan). KBLI yang perlu ada di NIB: 42101 (jalan), 42102 (jembatan).
**📊 Kualifikasi (confidence 0.88):** **Kecil (K)** cocok untuk awal — pagu paket s.d. ±Rp 2,5 M.
**📄 Dokumen kunci:** Akta + NIB + NPWP + Domisili + Laporan Keuangan + 1 PJTBU + 1 PJSKBU per subklas + SKK Ahli Muda Jalan & Jembatan.
**🎓 Personel:** Anda perlu **2 PJSKBU** (boleh 1 orang merangkap 2 subklas asalkan SKK keduanya aktif) + **1 PJTBU**. `[ASUMSI: tim teknis Anda sudah punya SKK; bila belum, sertifikasi via LSP dulu — saya bisa bantu peta jalannya.]`
*(QR: Hitung biaya · Buat checklist · Cek SKK saya · Asesmen kesiapan)*
> 

### Skenario B — Tambah subklasifikasi BG pada BUJK Menengah

> 👤 *"PT saya sudah punya SBU BS, sekarang mau tambah BG009 (bangunan komersial). BUJK M1. Apa langkahnya?"*
> 

> 🤖 **[smart_mapping + checklist_docs + cost_estimate]** *(→ MAPPER + DOCS + COST paralel)*
**Use case:** *tambah_sub*. **Subklas baru:** BG009.
📄 Dokumen tambahan minimum: bukti pengalaman BG (1 proyek sejenis untuk M1) + SKK Ahli Madya Bangunan Gedung + peralatan minimum sesuai BG009.
💰 Estimasi biaya tambah-sub M1: **Rp 8–14 jt** (tarif LSBU + admin). ⏱️ Timeline: **14–30 hari kerja** sejak dossier lengkap.
*(QR: Lihat detail biaya · Cek SKK calon PJSKBU BG · Lanjut buat surat)*
> 

### Skenario C — Perpanjangan SBU IL kualifikasi B1

> 👤 *"SBU instalasi listrik kami habis 2 bulan lagi."*
> 

> 🤖 **[perpanjangan_path]** *(→ DOCS + COMPLY + COST paralel)*
