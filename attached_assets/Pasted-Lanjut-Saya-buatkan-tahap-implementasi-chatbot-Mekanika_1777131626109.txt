# 21. Desain Knowledge Base Chatbot

Knowledge base adalah pusat data yang akan dibaca chatbot.

## Format data ideal

```json
{
  "id": "MEK-001",
  "klasifikasi": "Mekanikal",
  "subklasifikasi": "Teknik Tata Udara dan Refrigasi",
  "jabatan_kerja": "Ahli Perencanaan Sistem Tata Udara",
  "jenjang_jabatan": "Ahli Muda Perencanaan Sistem Tata Udara",
  "kualifikasi": "Ahli",
  "jenjang_kkni": 7,
  "standar_kompetensi": "SKKNI 131-2015",
  "status_skk": "Tersedia",
  "link_skk": "https://skkni-api.kemnaker.go.id/v1/public/documents/419f0d1b-f65b-447c-be03-8f226fac6a81/download",
  "kata_kunci": [
    "tata udara",
    "refrigasi",
    "hvac",
    "ac",
    "air conditioning",
    "perencana tata udara"
  ]
}
```

## Field penting

| Field                | Fungsi                                               |
| -------------------- | ---------------------------------------------------- |
| `id`                 | Kode unik data                                       |
| `klasifikasi`        | Bidang utama, yaitu Mekanikal                        |
| `subklasifikasi`     | Kelompok bidang, misalnya Plumbing, HVAC, Alat Berat |
| `jabatan_kerja`      | Nama jabatan utama                                   |
| `jenjang_jabatan`    | Level jabatan spesifik                               |
| `kualifikasi`        | Operator, Teknisi/Analis, atau Ahli                  |
| `jenjang_kkni`       | Level KKNI                                           |
| `standar_kompetensi` | SKKNI, SKK Khusus, atau status penyusunan            |
| `status_skk`         | Tersedia / Dalam Proses / Belum Ada                  |
| `link_skk`           | Link dokumen SKK/SKKNI                               |
| `kata_kunci`         | Sinonim pencarian user                               |

# 22. Kategori status SKK

Agar chatbot tidak salah menjawab, buat status SKK seperti ini:

```text
1. Tersedia
2. Dalam proses penyusunan
3. SKK Khusus
4. Belum tersedia
5. Skema LPJK / Akan disusun
```

Contoh penerapan:

```json
{
  "jabatan_kerja": "Operator Mesin Bubut Logam",
  "standar_kompetensi": "SKEMA LPJK/Akan disusun SKKNI/ SKK Khusus",
  "status_skk": "Skema LPJK / Akan disusun",
  "link_skk": "-"
}
```

Jawaban chatbot:

```text
Jabatan Operator Mesin Bubut Logam tersedia dalam klasifikasi Mekanikal.

Namun, standar kompetensinya tercatat sebagai:
SKEMA LPJK / Akan disusun SKKNI / SKK Khusus.

Artinya, dokumen SKKNI belum tersedia dalam data ini.
```

# 24. Rule rekomendasi otomatis

Chatbot bisa memberi rekomendasi berdasarkan **jenis pekerjaan** dan **pengalaman user**.

## A. Jika user pekerja lapangan pemula

```text
Kata kunci:
pemula, baru kerja, helper, asisten, belum punya sertifikat

Arahkan ke:
- Operator Level 1
- Operator Level 2
- Tukang
- Pelaksana dasar
```

## B. Jika user teknisi

```text
Kata kunci:
teknisi, maintenance, perawatan, inspeksi, troubleshooting

Arahkan ke:
- Teknisi/Analis
- KKNI Level 4 atau 5
```

## C. Jika user pengawas

```text
Kata kunci:
pengawas, supervisor, mandor, koordinator lapangan

Arahkan ke:
- Pengawas
- Mandor
- Teknisi/Analis KKNI 4, 5, atau 6
```

## D. Jika user engineer / ahli

```text
Kata kunci:
engineer, ahli, perencana, pemeriksa, pengkaji, konsultan

Arahkan ke:
- Ahli Muda KKNI 7
- Ahli Madya KKNI 8
- Ahli Utama KKNI 9
```

```text
1. [Nama Jabatan]
   - Subklasifikasi:
   - Kualifikasi:
   - Jenjang KKNI:
   - Standar Kompetensi:
   - Link SKK/SKKNI:

Alasan:
Jabatan ini sesuai karena Anda menyebut pengalaman di bidang [bidang user].

Catatan:
Jika pengalaman Anda masih dasar, pilih jenjang paling rendah terlebih dahulu. Jika sudah menjadi pengawas atau engineer, pilih jenjang lebih tinggi sesuai persyaratan sertifikasi.
```

# 31. Pseudocode pencarian

```javascript
function jawabPertanyaan(userInput) {
  const query = normalisasi(userInput);

  const hasilJabatan = cariJabatan(query);
  const hasilKKNI = cariKKNI(query);
  const hasilSKK = cariStandarKompetensi(query);
  const hasilSinonim = cariSinonim(query);

  if (hasilJabatan.length === 1) {
    return formatDetail(hasilJabatan[0]);
  }

  if (hasilJabatan.length > 1) {
    return formatPilihan(hasilJabatan);
  }

  if (hasilKKNI.length > 0) {
    return formatDaftarKKNI(hasilKKNI);
  }

  if (hasilSKK.length > 0) {
    return formatDaftarSKK(hasilSKK);
  }

  if (hasilSinonim.length > 0) {
    return formatPilihan(hasilSinonim);
  }

  return fallback();
}
```

# 34. Prompt final siap pakai

Ini bisa langsung dipakai untuk Custom GPT, chatbot website, atau WhatsApp bot berbasis AI.

```text
Anda adalah MekaSKK Bot, asisten informasi jabatan kerja bidang Mekanikal berbasis SKK/SKKNI.

Anda menjawab berdasarkan database klasifikasi Mekanikal yang diberikan.

Database memuat:
- Subklasifikasi
- Jabatan kerja
- Jenjang jabatan kerja
- Kualifikasi
- Jenjang KKNI
- Standar kompetensi kerja
- Link SKK/SKKNI

Aturan utama:
1. Jawab hanya berdasarkan data.
2. Jangan mengarang nomor SKKNI, jenjang KKNI, atau link.
3. Jika data tidak ditemukan, sampaikan bahwa data belum ditemukan.
4. Jika link SKK tidak tersedia, tulis "Link belum tersedia".
5. Jika standar masih dalam proses penyusunan, tulis sesuai data.
6. Jika user menyebut istilah umum, tampilkan beberapa jabatan yang relevan.
7. Jika user meminta rekomendasi, tanyakan atau gunakan informasi pekerjaan, pengalaman, dan bidangnya.
8. Gunakan bahasa Indonesia yang sopan, jelas, dan praktis.
9. Beri saran jenjang berdasarkan kualifikasi:
   - Operator: KKNI 1 sampai 3
   - Teknisi/Analis: KKNI 4 sampai 6
   - Ahli: KKNI 7 sampai 9
10. Jangan menjawab bidang di luar Mekanikal kecuali untuk menjelaskan bahwa data tidak tersedia.

Format jawaban detail:
Jabatan:
Subklasifikasi:
Jenjang Jabatan:
Kualifikasi:
Jenjang KKNI:
Standar Kompetensi:
Link SKK/SKKNI:
Catatan:

Jika ada banyak hasil:
Tampilkan maksimal 5 hasil teratas, lalu minta user memilih salah satu.
```
