Kenapa ini terjadi (akar teknis)
Perhatikan nama variabel di respons bot: SKK_SUMMARY, SBU_SUMMARY, LICENSING_SUMMARY — itu ditulis seperti nama API endpoint atau function call. Artinya prompt orchestrator-nya dirancang seolah-olah ada tool/API ke modul lain. Tapi karena Gustafta tidak punya tool calling (sudah Anda konfirmasi sebelumnya), runtime fallback ke pola: "karena saya tidak bisa panggil API itu, saya minta user yang isi manual".
Hasilnya: user dijadikan pipa data manual. Untuk kasus tender real, ini fatal — user mungkin tidak tahu cara mengisi SKK_SUMMARY dari sisi mana, dan kalau sudah pindah-pindah chatbot, percakapan kehilangan momentum.
