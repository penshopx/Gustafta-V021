## 4️⃣ KB-AHSP — Analisis Harga Satuan Pekerjaan

### Struktur

```json
{
	"ahsp_id": "AHSP-BG-2.1.1",
	"divisi": "Bangunan Gedung",
	"pekerjaan": "Pasangan bata merah 1:4, tebal 1/2 bata",
	"satuan": "m²",
	"komponen": {
		"bahan": [{"item": "bata merah", "koef": 70, "sat": "bh", "harga_ref": 1200}],
		"upah": [{"item": "tukang batu", "koef": 0.25, "sat": "OH", "harga_ref": 150000}],
		"alat": []
	},
	"overhead_profit": 0.15,
	"harga_satuan_ref": 158400,
	"region": "Jakarta",
	"sumber": "Permen PUPR 1/2022 Lampiran",
	"valid_until": "2025-12-31"
}
```

### Divisi Coverage

- Bangunan Gedung — 180 items
- Bangunan Sipil (Jalan, Jembatan, Drainase) — 220 items
- Mekanikal Elektrikal — 90 items
- Lansekap & Site Development — 60 items
- Pekerjaan Khusus (pile, bored, dewatering) — 50 items

**Harga referensi regional:** Jabodetabek, Bandung, Surabaya, Medan, Makassar, Balikpapan. Faktor regional 0.85–1.25 dari Jakarta baseline.

## 🔌 Cara Inject KB ke Agen

<aside>
🔌

Per agen, di blok `## KNOWLEDGE BASE` system prompt, tambahkan: `Akses KB-REG, KB-PROC, KB-DOC via internal lookup. Saat tidak yakin, jawab dengan ABD-3 [ASUMSI: ...] + confidence < 70 dan rekomendasikan verifikasi.` KB di-host sebagai (a) RAG vectorstore, (b) structured database Notion, atau (c) static lookup di prompt. Untuk MVP TENDERA, **gunakan opsi (b)** — database Notion per modul dengan property `agen_konsumen`.

</aside>
