# Buku II — KOLABORASI: the model the Teams/Organization Builder must honor

The Phase-3 "AI Organization / Teams" layer is the product implementation of this ebook. When designing any Tahap, check the design against these concepts.

## The single pattern that binds every chapter
Manusia → (niat & batas) → Tim Agen Domain → (usulan & ringkasan) → Manusia → ◆ gerbang manusia → Keputusan/Pertimbangan → Hasil berlipat.
Humans don't disappear; they move "up an orbit" from operator to intent-giver + gate. Agents return time, then wait for human judgment.

## Tujuh Prinsip KOLABORASI (universal, from the epilog)
1. Start from the repetitive, not the strategic — fold the boring first.
2. Agents must have a voice — cold corporate tone is rejected; a voice that knows you is trusted.
3. Human gate (◆) is non-negotiable for: decisions about humans (jobs, health, law, relationships), big money/contracts, and irreversible actions.
4. Three agents are enough at the start — more = complexity without power.
5. Automatic log + summary — without a trail, hybrid teams fall into chaos.
6. Honest escalation — a good agent knows when to stop and call a human.
7. New victory metric — not hours worked or raw output, but time returned, restored human touch, elevated judgment.

## Most design-relevant chapters
- **Bab 3 (Maya — Workflow Hybrid)**: 4 operan patterns = (1) AI-first+human review, (2) human-first+AI assist, (3) parallel exploration, (4) agent-to-agent with human gate. The fatal mistake = drawing a workflow with NO explicit decision gate. Every workflow needs ≥1 ◆ gate with a NAME, entry criteria, exit criteria. 6 design rules: design backward from the outcome; mark ◆; separate repetitive(agent) vs decision(human); put humans at ETHICAL points (e.g. rejections), not just technical ones; measure outcome not headcount; provide an escalation/circuit-breaker path.
- **Bab 4 (Pak Wira — Rapat)**: meetings do 4 separate functions — (1) info-sync, (2) decision, (3) social/culture, (4) collective learning. Agents take over (1) and part of (4); humans keep (2) and (3). Pre-meeting agent > in-meeting agent. KPI = decision latency, not duration. 5 meeting agents: Pre-Meeting Sync, Decision Brief, Decision Logger, Commitment Tracker, Retrospective Prep — they run at different times, never all in one meeting.
- **Bab 5 (Adi — Manajemen Tim Hybrid)**: manager roles 1-3 (task distributor, info filter, technical reviewer) shrink; roles 4-5 (career mentor, culture keeper) grow; 3 NEW roles appear — team-agent architect, hybrid-accountability keeper, strategy→architecture translator. Manager = conductor: choose composition (how many humans/agents), set tempo, guard the soul. Accountability for an agent's output falls on the human who deployed it.
- **Bab 6 (Dr. Vania — Asisten Domain Pribadi)**: a professional's personal 4-role agent team template: 📚 Kurator (curate/summarize new domain knowledge), 📖 Standar (explain guideline/regulation changes as a diff), 🤔 Skeptis (devil's advocate, what's missed/risky), 🌿 Penerjemah (translate professional output to client/patient language). Generic AI fails pros due to: shallow domain context, uncurated sources, no narrow roles, ethics/accountability. Curated sources > more sources. Names Gustafta Builder explicitly as the no-code assembly platform.
- **Bab 7 (Pak Anto — EdTech/Bimbel)**: "offline loses only to offline + AI." Domain tutor agents built on the school's OWN curriculum + best teachers' voice; mandatory Socratic mode (never an answer-machine); auto session log + summary read by the human teacher next morning; ◆ gate keeps decisions-about-the-child with humans; agents raise teachers from correctors to mentors. Tiered pricing (in-person / in-person+agent / agent-only).
- **Bab 8 (Bu Ningsih — UMKM)**: WA-native stack, 3 agents (Pelanggan, Stok, Pembukuan); live inside WhatsApp (no new app/login); family voice not corporate; voice-note + photo input; ◆ gate for complaints, abnormal orders, price/menu changes; "three agents enough."

## How to apply to the Organization Builder
- Every team/org template should make the ◆ human-gate a first-class, named element (name + entry/exit criteria), not an afterthought — "workflow without a gate is escapism, not design."
- Favor narrow roles over one broad agent (MultiClaw principle: many narrow roles, not one wide agent).
- Default to ~3 starter roles; resist role bloat.
- Surface log/summary + escalation as built-in expectations of any assembled team.
- Frame success to the (non-technical, Indonesian) user in human terms: time returned, judgment elevated — not headcount cut.
